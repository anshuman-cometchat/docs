## Overview

The CometChatSearchBar component provides a search input with debounced search functionality, a clear button, and full keyboard accessibility. It's designed for filtering and searching content in lists, conversations, and other UI elements.

## Basic Usage

### Basic Usage

```typescript expandable
import { Component } from '@angular/core';
import { CometChatSearchBarComponent } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [CometChatSearchBarComponent],
  template: `
    <cometchat-search-bar
      [placeholderText]="'Search messages...'"
      (searchChanged)="onSearchChanged($event)"
    ></cometchat-search-bar>
  `
})
export class AppComponent {
  onSearchChanged(event: { value: string }): void {
    console.log('Search value:', event.value);
    // Perform search operation
  }
}
```

### With Initial Value

## Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `searchText` | `string` | `''` | Initial/controlled search text value |
| `placeholderText` | `string` | `'Search'` | Placeholder text for the search input |
| `debounceDelay` | `number` | `300` | Debounce delay in milliseconds before emitting search events |
| `ariaLabel` | `string` | `undefined` | Custom aria-label for the search input (overrides placeholderText) |

## Events

| Event | Payload Type | Description |
|-------|-------------|-------------|
| `searchChanged` | `{ value: string }` | Emitted when search text changes (debounced) |
| `clearClick` | `void` | Emitted when the clear button is clicked |
| `focusFirstListItem` | `void` | Emitted when ArrowDown is pressed from the search input, signaling the parent to move focus to the first list item |

### Debouncing Explanation

The search bar uses RxJS debouncing to prevent excessive search operations. When a user types, the component waits for the specified `debounceDelay` (default 300ms) after the last keystroke before emitting the `searchChanged` event. This improves performance by reducing the number of search operations, especially when searching against a backend API.

```typescript
// Example: Longer debounce for API searches
<cometchat-search-bar
  [debounceDelay]="500"
  [placeholderText]="'Search users...'"
  (searchChanged)="searchUsers($event)"
></cometchat-search-bar>
```

## Accessibility

The CometChatSearchBar component is fully WCAG 2.1 Level AA compliant with comprehensive keyboard accessibility and screen reader support.

### Keyboard Shortcuts

| Key | Action | Context |
|-----|--------|---------|
| **Tab** | Focus the search input | When navigating to the component |
| **Type** | Enter search text | When input is focused |
| **Escape** | Clear the search | When input is focused and has text |
| **Tab** | Focus the clear button | When clear button is visible |
| **Enter/Space** | Clear the search | When clear button is focused |

### ARIA Attributes

The component implements proper ARIA attributes for accessibility:

| Element | ARIA Attributes | Purpose |
|---------|----------------|---------|
| Container | `role="search"` | Identifies the search landmark region |
| Input | `role="searchbox"` | Identifies the input as a search field |
| Input | `aria-label` | Provides accessible label (uses `ariaLabel` prop or falls back to `placeholderText`) |
| Clear button | `aria-label="Clear search"` | Describes the button's purpose |
| Icons | `aria-hidden="true"` | Hides decorative icons from screen readers |

### Custom ARIA Labels

You can provide a custom aria-label for better accessibility:

```typescript
<cometchat-search-bar
  [placeholderText]="'Search...'"
  [ariaLabel]="'Search conversations by name or message content'"
  (searchChanged)="onSearchChanged($event)"
></cometchat-search-bar>
```

### Screen Reader Announcements

- The search input is announced as "searchbox" with the aria-label text
- The clear button is announced as "Clear search button"
- The search landmark region helps users navigate to the search functionality
- Decorative icons are hidden from screen readers with `aria-hidden="true"`

### Focus Management

- **Visible Focus Indicators**: Clear focus outlines using `--cometchat-primary-color`
- **Focus-Visible Support**: Focus indicators only appear for keyboard navigation, not mouse clicks
- **Tab Order**: Logical tab order from input to clear button
- **Focus Restoration**: Focus remains on input after clearing with Escape key

### Accessibility Best Practices

1. **Always provide meaningful labels**: Use the `ariaLabel` prop for descriptive labels
2. **Test with keyboard only**: Ensure all functionality is accessible without a mouse
3. **Test with screen readers**: Verify announcements with NVDA, JAWS, or VoiceOver
4. **Maintain focus visibility**: Don't override the default focus indicators
5. **Use semantic HTML**: The component uses native `<input type="search">` for better semantics

## Best Practices

1. **Debounce Delay**: Use a longer debounce delay (500-800ms) for API searches to reduce server load
2. **Clear Feedback**: Always handle the `clearClick` event to reset your search results
3. **Loading States**: Show loading indicators during search operations
4. **Empty States**: Display appropriate messages when no results are found
5. **Accessibility**: Ensure the search bar is keyboard accessible and has proper ARIA labels
6. **Performance**: Use the debounced `searchChanged` event instead of listening to raw input events
