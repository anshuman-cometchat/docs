The `CometChatContextMenu` component displays a list of contextual actions in a menu format. It intelligently handles overflow by showing a subset of actions directly and placing additional actions in a submenu accessible via a "more" button. The component includes smart positioning logic to ensure menus stay within viewport boundaries.

## Overview

The ContextMenu component provides a flexible way to display contextual actions:

- **Smart Overflow**: Automatically splits actions into main menu and submenu based on `topMenuSize`
- **Intelligent Positioning**: Adapts placement based on available space (viewport, parent container, or centered)
- **Keyboard Accessible**: Full keyboard navigation with arrow keys, Enter, and Escape
- **ARIA Compliant**: Proper roles and attributes for screen readers
- **Focus Management**: Automatic focus handling and trapping within submenu
- **Customizable**: CSS variable-based styling for easy theming

## Basic Usage

### Simple Context Menu

```html
<cometchat-context-menu 
  [data]="menuItems"
  [topMenuSize]="3"
  (optionClick)="onOptionClick($event)">
</cometchat-context-menu>
```

```typescript expandable
import { CometChatOption } from '@cometchat/chat-uikit-angular';

@Component({

  imports: [CometChatContextMenuComponent],

  `,
  styles: [`

      display: flex;
      align-items: center;
      gap: 12px;
      padding: 12px;
      background: #f5f5f5;
      border-radius: 8px;
    }
    .message-content {
      flex: 1;
    }
  `]
})
export class MessageActionsComponent {
  messageActions: CometChatOption[] = [
    new CometChatOption({
      id: 'reply',
      title: 'Reply',
      iconURL: 'assets/icons/reply.svg'
    }),
    new CometChatOption({
      id: 'forward',
      title: 'Forward',
      iconURL: 'assets/icons/forward.svg'
    }),
    new CometChatOption({
      id: 'copy',
      title: 'Copy',
      iconURL: 'assets/icons/copy.svg'
    }),
    new CometChatOption({
      id: 'edit',
      title: 'Edit',
      iconURL: 'assets/icons/edit.svg'
    }),
    new CometChatOption({
      id: 'delete',
      title: 'Delete',
      iconURL: 'assets/icons/delete.svg'
    })
  ];

  handleAction(option: CometChatOption): void {
    console.log('Action selected:', option.title);
    
    switch (option.id) {
      case 'reply':
        this.replyToMessage();
        break;
      case 'forward':
        this.forwardMessage();
        break;
      case 'copy':
        this.copyMessage();

  replyToMessage(): void {

  }

  forwardMessage(): void {
    console.log('Forwarding message');
  }

  copyMessage(): void {
    console.log('Copying message');
  }

  editMessage(): void {
    console.log('Editing message');
  }

  deleteMessage(): void {
    console.log('Deleting message');
  }
}
```

## Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `data` | `ContextMenuItem[]` | `[]` | Array of menu items to display. Can be `CometChatOption`, `CometChatActionsIcon`, or `CometChatActionsView` |
| `topMenuSize` | `number` | `2` | Number of items visible in main menu before "more" button. Actual visible items = `topMenuSize - 1` to leave room for more button |
| `moreIconHoverText` | `string` | `undefined` | Tooltip text for the "more" button |
| `placement` | `Placement` | `Placement.left` | Menu placement position: `left`, `right`, `top`, or `bottom` |
| `closeOnOutsideClick` | `boolean` | `false` | Close submenu when clicking outside |
| `disableBackgroundInteraction` | `boolean` | `false` | Show overlay and disable background interaction when submenu is open |
| `useParentContainer` | `boolean` | `false` | Use parent element as viewport for positioning calculations |
| `useParentHeight` | `boolean` | `false` | Use parent height for positioning calculations |
| `forceStaticPlacement` | `boolean` | `false` | Force static placement without intelligent positioning logic |

## Events

| Event | Payload Type | Description |
|-------|-------------|-------------|
| `optionClick` | `EventEmitter<ContextMenuItem>` | Emitted when a menu option is clicked. Provides the clicked menu item object |

## Behavior

### Menu Splitting Logic

The ContextMenu automatically splits items into main menu and submenu:

- **Main Menu**: Shows first `topMenuSize - 1` items directly
- **More Button**: Shown when there are more items than `topMenuSize - 1`
- **Submenu**: Contains remaining items from `topMenuSize - 1` onwards

Example with `topMenuSize="3"` and 5 items:
- Main menu shows: Item 1, Item 2, More button
- Submenu contains: Item 3, Item 4, Item 5

### Positioning Strategies

The component supports three positioning strategies:

#### 1. Viewport Positioning (Default)

```typescript
// Uses window viewport as boundary
<cometchat-context-menu 
  [data]="items"
  [placement]="'left'">
</cometchat-context-menu>
```

#### 2. Parent Container Positioning

```typescript
// Uses nearest parent with 'cometchat' class as boundary
<cometchat-context-menu 
  [data]="items"
  [useParentContainer]="true"
  [placement]="'bottom'">
</cometchat-context-menu>
```

#### 3. Centered Positioning

```typescript
// Centers menu relative to trigger button
<cometchat-context-menu 
  [data]="items"
  [useParentHeight]="true"
  [placement]="'bottom'">
</cometchat-context-menu>
```

#### 4. Static Positioning

```typescript
// Forces exact placement without intelligent adjustments
<cometchat-context-menu 
  [data]="items"
  [forceStaticPlacement]="true"
  [placement]="'left'">
</cometchat-context-menu>
```

### Keyboard Navigation

The ContextMenu provides comprehensive keyboard support for full accessibility:

| Key | Action | Context |
|-----|--------|---------|
| `Tab` | Navigate to/from more button | Moves focus to the more button or next focusable element |
| `Shift + Tab` | Navigate backwards | Moves focus to previous focusable element |
| `Enter` | Activate | Opens submenu (on more button) or selects item (in submenu) |
| `Space` | Activate | Opens submenu (on more button) or selects item (in submenu) |
| `Arrow Down` | Navigate down | Moves focus to next item in submenu (wraps to first) |
| `Arrow Up` | Navigate up | Moves focus to previous item in submenu (wraps to last) |
| `Escape` | Close | Closes submenu and returns focus to more button |

**Keyboard Navigation Features:**
- **Automatic Focus**: When submenu opens, focus automatically moves to the first item
- **Wrapping Navigation**: Arrow keys wrap around from last to first item and vice versa
- **Focus Restoration**: When submenu closes (via Escape or selection), focus returns to the more button
- **Tab Behavior**: Tab key closes the submenu and moves focus to the next element in the page
- **Visual Feedback**: Focused items are highlighted with clear visual indicators

## Accessibility

The ContextMenu component includes comprehensive accessibility features:

### ARIA Attributes

- **Menu Roles**: `role="menu"` for both main menu and submenu
- **Menu Items**: `role="menuitem"` for each action item
- **Button Role**: `role="button"` for the more button
- **Expanded State**: `aria-expanded` indicates submenu open/closed state
- **Popup Indicator**: `aria-haspopup="true"` on more button
- **Selection State**: `aria-selected` for focused items
- **Labels**: `aria-label` on menu and items for screen reader context

### Screen Reader Support

Screen readers will announce:
- "Context menu" when focusing the component
- "More options, button, collapsed/expanded" for the more button
- Each action's title as users navigate
- "Selected" when an action is activated

### Focus Management

- **Initial Focus**: Automatically set to first submenu item when opened
- **Focus Trapping**: Arrow keys cycle through submenu items
- **Focus Restoration**: When closed, focus returns to more button
- **Tab Behavior**: Tab key closes submenu and moves to next focusable element

### Visual Focus Indicators

Clear focus indicators for keyboard navigation:
- 2px solid outline in primary color
- Inset outline (-2px offset) for submenu items
- Background highlight on focus
- High contrast mode support with 3px outline

### Dynamic Actions Based on Permissions

```typescript expandable
import { Component, OnInit } from '@angular/core';
import { CommonModule } from '@angular/common';
import { CometChatContextMenuComponent } from '@cometchat/chat-uikit-angular';
import { CometChatOption } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-permission-based-menu',
  standalone: true,
  imports: [CommonModule, CometChatContextMenuComponent],
  template: `
    <div class="content-item">
      <p>{{ contentTitle }}</p>
      
      <cometchat-context-menu 
        [data]="availableActions"
        [topMenuSize]="3"
        (optionClick)="handleAction($event)">
      </cometchat-context-menu>
    </div>
  `
})
export class PermissionBasedMenuComponent implements OnInit {
  contentTitle = 'Important Document';
  availableActions: CometChatOption[] = [];
  
  private allActions: CometChatOption[] = [
    new CometChatOption({
      id: 'view',
      title: 'View',
      iconURL: 'assets/icons/view.svg'
    }),
    new CometChatOption({
      id: 'download',
      title: 'Download',
      iconURL: 'assets/icons/download.svg'
    }),
    new CometChatOption({
      id: 'edit',
      title: 'Edit',
      iconURL: 'assets/icons/edit.svg'
    }),
    new CometChatOption({
      id: 'share',
      title: 'Share',
      iconURL: 'assets/icons/share.svg'
    }),
    new CometChatOption({
      id: 'delete',
      title: 'Delete',
      iconURL: 'assets/icons/delete.svg'
    })
  ];

  ngOnInit(): void {
    this.updateAvailableActions();
  }

  updateAvailableActions(): void {
    const permissions = this.getUserPermissions();
    
    this.availableActions = this.allActions.filter(action => {
      switch (action.id) {
        case 'view':
        case 'download':
          return true; // Everyone can view and download
        case 'edit':
          return permissions.canEdit;
        case 'share':
          return permissions.canShare;
        case 'delete':
          return permissions.canDelete;
        default:
          return false;
      }
    });
  }

  getUserPermissions() {
    // Mock permissions - replace with actual permission check
    return {
      canEdit: true,
      canShare: true,
      canDelete: false
    };
  }

  handleAction(option: CometChatOption): void {
    console.log('Action:', option.id);
  }
}
```

## Best Practices

<Tip>
  Set `topMenuSize` to show the most frequently used actions directly in the main menu for quick access.
</Tip>

<Warning>
  The actual number of visible main menu items is `topMenuSize - 1` to leave room for the "more" button. Plan accordingly.
</Warning>

<Info>
  Icons are displayed using CSS mask, which allows them to inherit the icon color from CSS variables. Use SVG icons for best results.
</Info>

<Tip>
  Use `disableBackgroundInteraction` when you want to ensure users focus on the menu without accidentally interacting with background elements.
</Tip>

<Warning>
  When using `useParentContainer`, ensure the parent element has the `cometchat` class for proper boundary detection.
</Warning>

<Tip>
  Enable `closeOnOutsideClick` for better UX when the menu should dismiss automatically when users click elsewhere.
</Tip>
