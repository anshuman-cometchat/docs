The `CometChatConfirmDialog` component provides a modal dialog for confirming important or destructive actions. It includes a title, message text, and action buttons (confirm/cancel) with built-in loading and error state management.

## Overview

The ConfirmDialog component is designed for user confirmation workflows:

- **Modal Overlay**: Displays as a centered modal with backdrop
- **Loading States**: Built-in loading animation on confirm button
- **Error Handling**: Displays error messages when operations fail
- **Accessibility**: Full keyboard support with focus trapping and restoration
- **Customizable**: CSS variable-based styling for easy theming
- **Localized**: Default text uses CometChat localization system

## Complete Example

```typescript expandable
import { Component, ViewChild } from '@angular/core';
import { CommonModule } from '@angular/common';
import { CometChatConfirmDialogComponent } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-delete-conversation',
  standalone: true,
  imports: [CommonModule, CometChatConfirmDialogComponent],
  template: `
    @if (showDialog) {
      <div class="dialog-backdrop">
        <cometchat-confirm-dialog 
          [title]="'Delete Conversation?'" 
          [messageText]="'This action cannot be undone. All messages will be permanently deleted.'"
          [confirmButtonText]="'Delete'"
          [cancelButtonText]="'Cancel'"
          (confirmClick)="handleConfirm()"
          (cancelClick)="handleCancel()">
        </cometchat-confirm-dialog>
      </div>
    }
    
    <button (click)="showDialog = true">
      Delete Conversation
    </button>
  `,
  styles: [`
    .dialog-backdrop {
      position: fixed;
      top: 0;
      left: 0;
      right: 0;
      bottom: 0;
      background: rgba(0, 0, 0, 0.5);
      display: flex;
      align-items: center;
      justify-content: center;
      z-index: 1000;
    }
  `]
})
export class DeleteConversationComponent {
  @ViewChild(CometChatConfirmDialogComponent) 
  dialog!: CometChatConfirmDialogComponent;
  
  showDialog = false;

  async handleConfirm(): Promise<void> {
    try {
      // Perform async operation
      await this.deleteConversation();
      
      // Notify dialog of success
      this.dialog.setSuccess();
      
      // Close dialog
      this.showDialog = false;
    } catch (error) {
      // Notify dialog of error
      this.dialog.setError();
    }
  }

  handleCancel(): void {
    this.showDialog = false;
  }

  private async deleteConversation(): Promise<void> {
    // Simulate API call
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        // Simulate success or failure
        Math.random() > 0.3 ? resolve() : reject(new Error('Failed to delete'));
      }, 2000);
    });
  }
}
```

## Events

| Event | Payload Type | Description |
|-------|-------------|-------------|
| `confirmClick` | `EventEmitter<void>` | Emitted when the confirm button is clicked. Parent should handle the action and call `setSuccess()` or `setError()` |
| `cancelClick` | `EventEmitter<void>` | Emitted when the cancel button is clicked or Escape key is pressed |

## Methods

The component exposes two methods for managing loading and error states:

### setSuccess()

Call this method after successfully completing the confirmed action:

```typescript
this.dialog.setSuccess();
```

This will:
- Stop the loading animation
- Clear any error messages
- Reset the dialog to its initial state

### setError()

Call this method if the confirmed action fails:

```typescript
this.dialog.setError();
```

This will:
- Stop the loading animation
- Display a localized error message
- Keep the dialog open for retry

## Behavior

### Loading State

When the confirm button is clicked:
1. The `confirmClick` event is emitted
2. The confirm button enters loading state automatically
3. The button is disabled during loading
4. Parent component performs the async operation
5. Parent calls `setSuccess()` or `setError()` based on result

```typescript expandable
async handleConfirm(): Promise<void> {
  try {
    await this.performAction();
    this.dialog.setSuccess();
    this.closeDialog();
  } catch (error) {
    this.dialog.setError();
    // Dialog stays open for user to retry or cancel
  }
}
```

### Error State

When `setError()` is called:
- A localized error message appears below the message text
- The loading state is cleared
- The dialog remains open
- User can retry (confirm again) or cancel

### Focus Management

The dialog implements comprehensive focus management:
- **Initial Focus**: Automatically set to the cancel button when opened
- **Focus Trapping**: Tab key cycles between cancel and confirm buttons
- **Focus Restoration**: When closed, focus returns to the element that opened the dialog
- **Escape Key**: Closes the dialog and triggers `cancelClick` event

## Accessibility

The ConfirmDialog component includes comprehensive accessibility features:

### Keyboard Support

| Key | Action |
|-----|--------|
| `Tab` | Moves focus between cancel and confirm buttons |
| `Shift+Tab` | Moves focus backward between buttons |
| `Enter` | Activates the focused button |
| `Space` | Activates the focused button |
| `Escape` | Closes the dialog (triggers cancelClick) |

### ARIA Attributes

- **role="dialog"**: Identifies the element as a dialog
- **aria-modal="true"**: Indicates the dialog is modal
- **aria-labelledby**: Points to the title element for screen reader context
- **aria-disabled**: Set on buttons during loading state

### Screen Reader Support

Screen readers will announce:
- "Dialog, [title]" when the dialog opens
- Button labels and states as users navigate
- Error messages when they appear
- Loading state on the confirm button

### Focus Indicators

- 2px solid outline in primary color
- Visible only during keyboard navigation (`:focus-visible`)

## Examples

### Delete Confirmation with Retry

```typescript expandable
import { Component, ViewChild } from '@angular/core';
import { CometChatConfirmDialogComponent } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-delete-message',
  standalone: true,
  imports: [CometChatConfirmDialogComponent],
  template: `
    @if (showDeleteDialog) {
      <div class="dialog-backdrop" (click)="handleCancel()">
        <div (click)="$event.stopPropagation()">
          <cometchat-confirm-dialog 
            [title]="'Delete Message?'" 
            [messageText]="'This message will be deleted for everyone.'"
            [confirmButtonText]="'Delete'"
            [cancelButtonText]="'Cancel'"
            (confirmClick)="handleDelete()"
            (cancelClick)="handleCancel()">
          </cometchat-confirm-dialog>
        </div>
      </div>
    }
  `,
  styles: [`
    .dialog-backdrop {
      position: fixed;
      top: 0;
      left: 0;
      right: 0;
      bottom: 0;
      background: rgba(0, 0, 0, 0.5);
      display: flex;
      align-items: center;
      justify-content: center;
      z-index: 1000;
    }
  `]
})
export class DeleteMessageComponent {
  @ViewChild(CometChatConfirmDialogComponent) 
  dialog!: CometChatConfirmDialogComponent;
  
  showDeleteDialog = false;
  retryCount = 0;
  maxRetries = 3;

  async handleDelete(): Promise<void> {
    try {
      await this.deleteMessage();
      this.dialog.setSuccess();
      this.showDeleteDialog = false;
      this.retryCount = 0;
    } catch (error) {
      this.retryCount++;
      
      if (this.retryCount >= this.maxRetries) {
        console.error('Max retries reached');
        this.dialog.setError();
      } else {
        // Retry automatically
        setTimeout(() => this.handleDelete(), 1000);
      }
    }
  }

  handleCancel(): void {
    this.showDeleteDialog = false;
    this.retryCount = 0;
  }

  private async deleteMessage(): Promise<void> {
    // API call to delete message
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        Math.random() > 0.5 ? resolve() : reject(new Error('Network error'));
      }, 1500);
    });
  }
}
```

import { Component, ViewChild } from '@angular/core';

## Best Practices

<Tip>
  Always use ViewChild to get a reference to the dialog component so you can call `setSuccess()` or `setError()` methods.
</Tip>

<Warning>
  Don't forget to call `setSuccess()` or `setError()` after handling the confirm action. The loading state will persist until one of these methods is called.
</Warning>

<Info>
  The dialog automatically manages focus and keyboard accessibility. You don't need to manually handle focus restoration.
</Info>

<Tip>
  Use the error state to allow users to retry failed operations without closing and reopening the dialog.
</Tip>

<Warning>
  Remember to wrap the dialog in a backdrop element with proper positioning and z-index to ensure it appears above other content.
</Warning>
