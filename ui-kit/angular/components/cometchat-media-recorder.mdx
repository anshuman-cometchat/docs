## Overview

The `CometChatMediaRecorder` component provides audio recording functionality with permission handling, recording controls, and audio preview. It supports start, pause, resume, and stop operations with a built-in timer display.

## Basic Usage

### Basic Usage

```typescript expandable
import { Component } from '@angular/core';
import { CometChatMediaRecorderComponent } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [CometChatMediaRecorderComponent],
  template: `
    <cometchat-media-recorder
      (submitRecording)="onSubmitRecording($event)"
      (closeRecording)="onCloseRecording()"
      (recordingError)="onRecordingError($event)"
    ></cometchat-media-recorder>
  `
})
export class AppComponent {
  onSubmitRecording(blob: Blob): void {
    console.log('Recording submitted:', blob);
    // Handle the recorded audio blob
  }

  onCloseRecording(): void {
    console.log('Recording closed');
  }

  onRecordingError(error: Error): void {
    console.error('Recording error:', error);
  }
}
```

### Auto-Recording

```typescript
<cometchat-media-recorder
  [autoRecording]="true"
  (submitRecording)="onSubmitRecording($event)"
></cometchat-media-recorder>
```

## Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `autoRecording` | `boolean` | `false` | Automatically start recording when component mounts |

## Events

| Event | Payload Type | Description |
|-------|-------------|-------------|
| `closeRecording` | `void` | Emitted when the close button is clicked |
| `submitRecording` | `Blob` | Emitted when recording is submitted with the recorded audio blob |
| `recordingError` | `Error` | Emitted when a recording error occurs (e.g., permission denied) |

## Features

### Recording Controls

The component provides intuitive controls for managing audio recording:

- **Start/Resume Button**: Begins recording or resumes from paused state
- **Pause Button**: Temporarily pauses the recording
- **Stop Button**: Stops recording and shows preview
- **Close Button**: Cancels recording and closes the recorder

### Timer Display

A real-time timer displays the recording duration in MM:SS format, helping users track their recording length.

### Permission Handling

The component automatically handles microphone permissions:

- Requests permission when recording starts
- Displays error message if permission is denied
- Monitors permission changes during recording
- Provides clear feedback to users about permission status

### Audio Preview

After stopping a recording, users can preview the audio using the native HTML audio player before submitting.

## Accessibility

The component is fully accessible and follows WCAG 2.1 Level AA guidelines:

### Keyboard Navigation

All recording controls are fully keyboard accessible:

| Key | Action | Context |
|-----|--------|---------|
| `Tab` | Navigate between buttons | Move focus through all interactive elements |
| `Shift + Tab` | Navigate backwards | Move focus in reverse order |
| `Enter` | Activate button | Start/pause/stop/close recording or submit |
| `Space` | Activate button | Start/pause/stop/close recording or submit |
| `Escape` | Cancel recording | Close recorder and reset state (works from anywhere in component) |

**Keyboard Navigation Flow:**
1. Tab to Start button → Press Enter/Space to begin recording
2. Tab to Pause button → Press Enter/Space to pause
3. Tab to Stop button → Press Enter/Space to stop and preview
4. Tab to Submit button → Press Enter/Space to submit recording
5. Press Escape at any time to cancel and close

### Screen Reader Support

The component provides comprehensive screen reader support:

- **Live Regions**: Recording state changes are announced automatically
  - "Recording" when recording starts
  - "Paused" when recording is paused
  - "Ready to record" when in idle state
- **Button Labels**: All buttons have descriptive `aria-label` attributes
  - "Start recording" / "Resume recording"
  - "Pause recording"
  - "Stop recording"
  - "Close recorder"
  - "Submit recording"
- **Timer**: Recording duration is labeled as "Recording duration"
- **Audio Preview**: Preview player is labeled as "Audio preview"
- **Error Messages**: Permission errors use `role="alert"` for immediate announcement

## Advanced Usage

### Handling Recording Submission

```typescript expandable
import { Component } from '@angular/core';

@Component({
  selector: 'app-chat',
  template: `
    <cometchat-media-recorder
      (submitRecording)="handleRecording($event)"
    ></cometchat-media-recorder>
  `
})
export class ChatComponent {
  async handleRecording(blob: Blob): Promise<void> {
    // Convert blob to File
    const file = new File([blob], 'recording.webm', { type: blob.type });
    
    // Upload to server
    const formData = new FormData();
    formData.append('audio', file);
    
    try {
      await fetch('/api/upload-audio', {
        method: 'POST',
        body: formData
      });
      console.log('Recording uploaded successfully');
    } catch (error) {
      console.error('Upload failed:', error);
    }
  }
}
```

### Error Handling

```typescript expandable
@Component({
  selector: 'app-chat',
  template: `
    <cometchat-media-recorder
      (recordingError)="handleError($event)"
    ></cometchat-media-recorder>
    
    @if (errorMessage) {
      <div class="error">
        {{ errorMessage }}
      </div>
    }
  `
})
export class ChatComponent {
  errorMessage = '';

  handleError(error: Error): void {
    if (error.name === 'NotAllowedError') {
      this.errorMessage = 'Microphone access denied. Please enable it in browser settings.';
    } else if (error.name === 'NotFoundError') {
      this.errorMessage = 'No microphone found. Please connect a microphone.';
    } else {
      this.errorMessage = 'Recording error occurred. Please try again.';
    }
  }
}
```

## Browser Compatibility

The component uses the MediaRecorder API, which is supported in:

- Chrome 47+
- Firefox 25+
- Safari 14.1+
- Edge 79+

**Note**: Microphone access requires HTTPS in production environments.

## Permission Requirements

The component requires microphone permission to function. Users will see a browser permission prompt when recording starts for the first time. If permission is denied, the component displays a helpful error message with instructions to enable microphone access.

## Best Practices

1. **Always handle the `recordingError` event** to provide user feedback
2. **Test microphone permissions** before showing the recorder
3. **Provide clear instructions** to users about recording controls
4. **Handle blob data appropriately** - convert to File or upload immediately
5. **Consider file size limits** for long recordings
6. **Provide visual feedback** during recording (the timer helps with this)
