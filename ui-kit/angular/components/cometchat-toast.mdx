The `CometChatToast` component provides a consistent, accessible toast notification system across the UIKit. It supports different types (success, error, warning, info) with auto-dismiss functionality, manual close options, and a programmatic service for creating toasts dynamically.

## Overview

The Toast component is designed for providing temporary feedback to users:

- **Multiple Types**: Success, error, warning, and info variants with appropriate icons
- **Auto-Dismiss**: Configurable duration for automatic dismissal
- **Manual Control**: Optional close button for user-controlled dismissal
- **Programmatic API**: Service-based creation for dynamic toast management
- **Toast Stacking**: Multiple toasts stack vertically with proper spacing
- **Accessibility**: ARIA live regions and keyboard support for screen readers
- **Customizable**: CSS variable-based styling for easy theming

## Basic Usage

### Simple Toast

```html
<cometchat-toast 
  [text]="'Operation completed successfully!'" 
  [type]="'success'"
  (toastClosed)="onToastClose()">
</cometchat-toast>
```

## Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `text` | `string` | `''` | The message text to display in the toast |
| `type` | `ToastType` | `ToastType.info` | Toast type for styling and icon: `'success'`, `'error'`, `'warning'`, or `'info'` |
| `duration` | `number` | `3000` | Duration in milliseconds before auto-dismiss. Set to `0` to disable auto-dismiss |
| `showCloseButton` | `boolean` | `true` | Whether to show the close button for manual dismissal |
| `dismissOnEscape` | `boolean` | `true` | Whether pressing Escape key dismisses the toast |

## Events

| Event | Payload Type | Description |
|-------|-------------|-------------|
| `toastClosed` | `EventEmitter<void>` | Emitted when the toast is closed (either automatically or manually) |

## Using the Toast Service

The `CometChatToastService` provides a programmatic way to create and manage toasts dynamically. This is the recommended approach for most use cases.

### Basic Service Usage

```typescript expandable
import { Component } from '@angular/core';
import { CometChatToastService } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-message-actions',
  standalone: true,
  template: `
    <div class="message-actions">
      <button (click)="sendMessage()">Send Message</button>
      <button (click)="deleteMessage()">Delete Message</button>
      <button (click)="copyMessage()">Copy Message</button>
    </div>
  `
})
export class MessageActionsComponent {
  constructor(private toastService: CometChatToastService) {}

  sendMessage(): void {
    // Simulate sending message
    setTimeout(() => {
      this.toastService.success('Message sent successfully!');
    }, 500);
  }

  deleteMessage(): void {
    // Simulate deleting message
    setTimeout(() => {
      this.toastService.error('Failed to delete message. Please try again.');
    }, 500);
  }

  copyMessage(): void {
    // Simulate copying message
    this.toastService.info('Message copied to clipboard');
  }
}
```

## Behavior

### Auto-Dismiss

Toasts automatically dismiss after the specified duration:

```typescript
// Auto-dismiss after 3 seconds (default)
this.toastService.success('Message sent!');

// Auto-dismiss after 5 seconds
this.toastService.error('Error occurred', { duration: 5000 });

// No auto-dismiss (persistent)
this.toastService.warning('Important notice', { duration: 0 });
```

### Manual Close

Users can manually close toasts using the close button:

```typescript
// With close button (default)
this.toastService.info('New notification', { showCloseButton: true });

// Without close button
this.toastService.success('Saved!', { showCloseButton: false });
```

### Toast Stacking

Multiple toasts stack vertically with 10px spacing:

```typescript
// Create multiple toasts
this.toastService.success('First operation complete');
this.toastService.info('Second operation in progress');
this.toastService.warning('Third operation needs attention');

// They will stack from top to bottom
```

### Toast Positioning

Toasts are positioned in the top-right corner by default:
- Fixed position at `top: 20px`, `right: 20px`
- Z-index of `9999` to appear above other content
- Stacked vertically with flexbox

## Accessibility

The Toast component includes comprehensive accessibility features to ensure WCAG 2.1 Level AA compliance:

### ARIA Attributes

The component automatically sets appropriate ARIA attributes based on toast type:

| Toast Type | ARIA Role | ARIA Live | Behavior |
|------------|-----------|-----------|----------|
| `success` | `status` | `polite` | Waits for screen reader pause before announcing |
| `info` | `status` | `polite` | Waits for screen reader pause before announcing |
| `warning` | `alert` | `assertive` | Interrupts screen reader for immediate announcement |
| `error` | `alert` | `assertive` | Interrupts screen reader for immediate announcement |

```typescript
// Success and info toasts use role="status" with aria-live="polite"
// They wait for a pause in screen reader activity before announcing
this.toastService.success('Message sent successfully');

// Warning and error toasts use role="alert" with aria-live="assertive"
// They interrupt screen reader activity for immediate announcement
this.toastService.error('Failed to send message');
```

### Keyboard Support

The Toast component provides full keyboard accessibility:

| Key | Action | Notes |
|-----|--------|-------|
| `Tab` | Moves focus to the close button (if visible) | Standard tab navigation |
| `Enter` | Closes the toast when close button is focused | Activates close button |
| `Space` | Closes the toast when close button is focused | Activates close button |
| `Escape` | Dismisses the toast immediately | Works from anywhere on the page |

```typescript
// Escape key dismissal is enabled by default
this.toastService.error('An error occurred');
// User can press Escape to dismiss

// Disable Escape key dismissal if needed
<cometchat-toast 
  [text]="'Important message'"
  [dismissOnEscape]="false">
</cometchat-toast>
```

### Screen Reader Announcements

The component provides appropriate announcements based on toast type:

```typescript expandable
// Error toast - announced immediately with "Alert" prefix
this.toastService.error('Failed to send message');
// Screen reader: "Alert: Failed to send message"

// Warning toast - announced immediately with "Alert" prefix
this.toastService.warning('Your session will expire soon');
// Screen reader: "Alert: Your session will expire soon"

// Success toast - announced politely with "Status" prefix
this.toastService.success('Message sent successfully');
// Screen reader: "Status: Message sent successfully"

// Info toast - announced politely with "Status" prefix
this.toastService.info('New messages available');
// Screen reader: "Status: New messages available"
```

### Focus Management

The Toast component follows accessibility best practices for focus management:

- **No Focus Trap**: Toast does not trap keyboard focus, allowing users to continue interacting with the page
- **Keyboard Accessible Close Button**: Close button is fully keyboard accessible with visible focus indicators
- **Focus Indicators**: Clear visual focus indicators meet WCAG contrast requirements (3:1 minimum)
- **Non-Blocking**: Toast appears as an overlay but doesn't block interaction with underlying content

```typescript
// Toast appears but doesn't steal focus
this.toastService.info('New notification');
// User can continue typing or interacting with the page

// Close button receives focus when tabbed to
// Visible focus outline appears for keyboard users
```

### API Error Handling

```typescript expandable
import { Component } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { CometChatToastService } from '@cometchat/chat-uikit-angular';
import { catchError } from 'rxjs/operators';
import { of } from 'rxjs';

@Component({
  selector: 'app-data-loader',
  standalone: true,
  template: `
    <div class="data-loader">
      <button (click)="loadData()">Load Data</button>
      @if (data) {
        <div>{{ data | json }}</div>
      }
    </div>
  `
})
export class DataLoaderComponent {
  data: any = null;

  constructor(
    private http: HttpClient,
    private toastService: CometChatToastService
  ) {}

  loadData(): void {
    this.toastService.info('Loading data...', { duration: 2000 });

    this.http.get('/api/data')
      .pipe(
        catchError(error => {
          this.toastService.error('Failed to load data. Please try again.', {
            duration: 5000,
            showCloseButton: true
          });
          return of(null);
        })
      )
      .subscribe(data => {
        if (data) {
          this.data = data;
          this.toastService.success('Data loaded successfully!', {
            duration: 3000
          });
        }
      });
  }
}
```

```typescript expandable
@Component({

export class FileUploaderComponent {
  selectedFile: File | null = null;

  constructor(private toastService: CometChatToastService) {}

  onFileSelect(event: any): void {
    this.selectedFile = event.target.files[0];
    if (this.selectedFile) {
      this.toastService.info(`File selected: ${this.selectedFile.name}`, {
        duration: 2000
      });
    }
  }

  async uploadFile(): Promise<void> {
    if (!this.selectedFile) return;

    const uploadToastId = this.toastService.info('Uploading file...', {
      duration: 0,
      showCloseButton: false
    });

    try {
      // Simulate upload
      await this.simulateUpload();
      
      this.toastService.remove(uploadToastId);
      this.toastService.success('File uploaded successfully!', {
        duration: 3000
      });
      
      this.selectedFile = null;
    } catch (error) {
      this.toastService.remove(uploadToastId);
      this.toastService.error('Upload failed. Please try again.', {
        duration: 5000,
        showCloseButton: true
      });
    }
  }

  private simulateUpload(): Promise<void> {
    return new Promise((resolve) => {
      setTimeout(resolve, 2000);
    });
  }
}
```

### Undo Actions

```typescript expandable
import { Component } from '@angular/core';
import { CometChatToastService } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-message-list',
  standalone: true,
  template: `
    <div class="message-list">
      @for (message of messages; track message.id) {
        <div class="message-item">
          <span>{{ message.text }}</span>
          <button (click)="deleteMessage(message)">Delete</button>
        </div>
      }
    </div>
  `
})
export class MessageListComponent {
  messages = [
    { id: 1, text: 'Hello!', deleted: false },
    { id: 2, text: 'How are you?', deleted: false },
    { id: 3, text: 'See you later!', deleted: false }
  ];

  private deletedMessage: any = null;
  private undoTimeout: any = null;

  constructor(private toastService: CometChatToastService) {}

  deleteMessage(message: any): void {
    this.deletedMessage = { ...message };
    const index = this.messages.findIndex(m => m.id === message.id);
    this.messages.splice(index, 1);

    const toastId = this.toastService.warning('Message deleted', {
      duration: 5000,
      showCloseButton: true
    });

    // Set undo timeout
    this.undoTimeout = setTimeout(() => {
      this.deletedMessage = null;
    }, 5000);
  }

  undoDelete(): void {
    if (this.deletedMessage) {
      this.messages.push(this.deletedMessage);
      this.deletedMessage = null;
      
      if (this.undoTimeout) {
        clearTimeout(this.undoTimeout);
      }
      
      this.toastService.info('Delete undone', { duration: 2000 });
    }
  }
}
```

## Best Practices

<Tip>
  Use the toast service for programmatic toast creation instead of manually managing toast components in templates.
</Tip>

<Warning>
  Avoid showing too many toasts simultaneously. Consider clearing old toasts or limiting the maximum number of active toasts.
</Warning>

<Info>
  Use appropriate toast types: success for confirmations, error for failures, warning for cautions, and info for general notifications.
</Info>

<Tip>
  For critical errors that require user action, use `duration: 0` to create persistent toasts with a close button.
</Tip>

<Warning>
  Keep toast messages concise and actionable. Long messages may be truncated or difficult to read.
</Warning>

<Info>
  Error and warning toasts use `aria-live="assertive"` for immediate screen reader announcements, while success and info toasts use `aria-live="polite"` to avoid interrupting users.
</Info>
