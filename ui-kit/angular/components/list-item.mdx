The `CometChatListItem` component is a versatile list item component that displays user or group information with customizable leading, trailing, title, subtitle, and menu views. It provides consistent styling and behavior for lists throughout your chat application.

## Overview

The ListItem component offers flexible customization options:

- **Avatar Display**: Automatically shows avatar with initials fallback
- **Template Projection**: Customize any part of the list item with templates
- **Hover States**: Shows menu view on hover, hiding trailing view
- **Keyboard Accessible**: Full keyboard navigation support
- **Click Handling**: Emits events with item ID for easy handling

## Basic Usage

### Simple List Item with Avatar

```html
<cometchat-list-item
  [id]="'user-123'"
  [avatarURL]="'https://example.com/avatar.jpg'"
  [avatarName]="'John Doe'"
  [title]="'John Doe'"
  (listItemClick)="onItemClick($event)">
</cometchat-list-item>
```

### Using with CometChat User Object

```typescript expandable
import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';
import { CometChat } from '@cometchat/chat-sdk-javascript';
import { CometChatListItemComponent } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-user-list',
  standalone: true,
  imports: [CommonModule, CometChatListItemComponent],
  template: `
    @for (user of users; track user.getUid()) {
      <cometchat-list-item
        [id]="user.getUid()"
        [avatarURL]="user.getAvatar()"
        [avatarName]="user.getName()"
        [title]="user.getName()"
        [subtitleView]="statusTemplate"
        (listItemClick)="onUserSelect($event)">
      </cometchat-list-item>
    }

    <ng-template #statusTemplate let-user>
      <span class="status">{{ user.getStatus() }}</span>
    </ng-template>
  `
})
export class UserListComponent {
  users: CometChat.User[] = [];

  ngOnInit() {
    this.loadUsers();
  }

  loadUsers() {
    const usersRequest = new CometChat.UsersRequestBuilder()
      .setLimit(30)
      .build();

    usersRequest.fetchNext().then(
      (users: CometChat.User[]) => {
        this.users = users;
      }
    );
  }

  onUserSelect(event: { id: string }) {
    console.log('Selected user:', event.id);
  }
}
```

## Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `id` | `string` | `''` | Unique identifier for the list item |
| `avatarURL` | `string` | `''` | URL of the avatar image |
| `avatarName` | `string` | `''` | Name for generating avatar initials |
| `title` | `string` | `''` | Primary text displayed in the list item |
| `subtitle` | `string` | `''` | Subtitle text for accessibility (used in aria-label) |
| `menuView` | `TemplateRef<any>` | `null` | Template for menu view (shown on hover/focus) |
| `subtitleView` | `TemplateRef<any>` | `null` | Template for subtitle content |
| `trailingView` | `TemplateRef<any>` | `null` | Template for trailing content (right side) |
| `titleView` | `TemplateRef<any>` | `null` | Template for custom title view |
| `leadingView` | `TemplateRef<any>` | `null` | Template for custom leading view (replaces avatar) |
| `stopEventPropagation` | `boolean` | `false` | Stop event propagation on trailing view click |
| `ariaLabel` | `string` | `undefined` | Custom ARIA label (overrides default title + subtitle) |
| `disableTabIndex` | `boolean` | `false` | Whether to disable the tabindex on the list item (when parent manages focus) |
| `isFocused` | `boolean` | `false` | Whether the list item is currently focused (managed by parent) |
| `menuShortcutKey` | `string \| null` | `'M'` | Keyboard shortcut key to toggle the menu view. Per WCAG 2.1.4, single-character shortcuts must be configurable |

## Events

| Event | Payload Type | Description |
|-------|-------------|-------------|
| `listItemClick` | `{ id: string }` | Emitted when the list item is clicked |

## Behavior

### Hover State Management

The component automatically manages hover states:

1. **Default State**: Shows `trailingView` on the right side
2. **Hover State**: Hides `trailingView` and shows `menuView` instead
3. **Mouse Leave**: Returns to default state

This provides a clean interface where contextual actions appear only when needed.

### Click Handling

```typescript expandable
@Component({
  selector: 'app-conversation-list',
  template: `
    <cometchat-list-item
      [id]="conversation.id"
      [title]="conversation.name"
      [stopEventPropagation]="true"
      (listItemClick)="openConversation($event)">
    </cometchat-list-item>
  `
})
export class ConversationListComponent {
  openConversation(event: { id: string }) {
    console.log('Opening conversation:', event.id);
    // Navigate to conversation or load messages
  }
}
```

### Event Propagation Control

Use `stopEventPropagation` to prevent clicks on trailing/menu views from triggering the list item click:

```html expandable
<cometchat-list-item
  [id]="user.id"
  [title]="user.name"
  [stopEventPropagation]="true"
  [trailingView]="actionButtonTemplate"
  (listItemClick)="selectUser($event)">
</cometchat-list-item>

<ng-template #actionButtonTemplate>
  <button (click)="performAction($event)">Action</button>
</ng-template>
```

## Accessibility

The ListItem component includes comprehensive accessibility features:

### Keyboard Navigation

The component is fully keyboard accessible with the following features:

- **Tab Navigation**: List item is focusable via Tab key (tabindex="0")
- **Enter/Space Activation**: Both Enter and Space keys activate the list item
- **Menu Access**: Press 'M' key to toggle menu view visibility
- **Nested Elements**: All nested interactive elements (buttons, links) are independently focusable
- **Logical Tab Order**: Tab order follows visual layout (parent list item, then nested elements)

### Keyboard Support

| Key | Action |
|-----|--------|
| `Tab` | Focus the list item |
| `Shift+Tab` | Focus previous element |
| `Enter` | Activate the list item (triggers click event) |
| `Space` | Activate the list item (triggers click event) |
| `M` | Toggle menu view visibility (when menuView is provided) |

### ARIA Attributes

The component implements proper ARIA attributes for screen reader support:

- **role="listitem"**: Identifies the element as a list item
- **aria-label**: Provides descriptive label combining title and subtitle
- **aria-describedby**: Links to subtitle element for additional context
- **role="group"**: Applied to menu view container
- **Custom Labels**: Support for custom aria-label via `ariaLabel` input

### Focus Management

- **Visible Focus Indicators**: Clear 2px outline in primary color with 2px offset
- **High Contrast Mode**: Enhanced 3px outline for better visibility
- **Focus-Visible**: Uses :focus-visible pseudo-class for keyboard-only focus indicators
- **Nested Element Focus**: Independent focus indicators for all nested interactive elements
- **Menu Persistence**: Menu view remains visible when list item has focus

### Nested Interactive Elements

The component properly handles nested interactive elements:

- **Independent Focusability**: All nested buttons, links, and elements with tabindex are independently focusable
- **Event Isolation**: Clicking nested elements doesn't trigger parent list item click
- **Proper Tab Order**: Nested elements follow logical tab order after parent
- **Focus Indicators**: Each nested element has its own visible focus indicator

### Screen Reader Support

- **Semantic HTML**: Uses proper HTML structure for screen readers
- **Descriptive Labels**: Combines title and subtitle for complete context
- **State Announcements**: Menu view has descriptive aria-label
- **Subtitle Association**: Subtitle is properly associated via aria-describedby

### Best Practices for Accessibility

<Tip>
  Always provide meaningful `title` and `subtitle` values for better screen reader context. The component automatically combines these for the aria-label.
</Tip>

<Warning>
  When using nested interactive elements (buttons, links), ensure they have proper labels and are keyboard accessible. The component preserves their focusability.
</Warning>

<Info>
  The menu view can be accessed via keyboard (M key) or mouse hover, ensuring all users can access contextual actions.
</Info>

## Best Practices

<Tip>
  Always provide an `id` property for proper event handling and tracking. This is especially important when working with lists of items.
</Tip>

<Warning>
  When using `menuView`, remember that it replaces `trailingView` on hover. Design your UI accordingly to avoid confusion.
</Warning>

<Info>
  Use `stopEventPropagation` when you have clickable elements in `trailingView` or `menuView` that shouldn't trigger the list item click event.
</Info>

<Tip>
  For better performance with large lists, use Angular's `@for` with `track` to help Angular identify which items have changed.
</Tip>
