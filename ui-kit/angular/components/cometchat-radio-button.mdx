The `CometChatRadioButton` component provides a consistent, accessible radio button implementation across the UIKit. It supports radio button grouping, keyboard navigation with arrow keys, and Angular forms integration via ControlValueAccessor.

## Overview

The Radio Button component is designed for single-selection patterns throughout your chat application:

- **Radio Grouping**: Automatically manages selection within named groups
- **Forms Integration**: Implements ControlValueAccessor for seamless Angular forms support
- **Keyboard Navigation**: Full keyboard support with Space, Arrow keys for group navigation
- **Accessibility**: ARIA attributes and screen reader support
- **Customizable**: CSS variable-based styling for easy theming
- **Responsive**: Hover and focus states with visual feedback

## Basic Usage

### Simple Radio Button

```html
<cometchat-radio-button 
  [name]="'size'"
  [value]="'small'"
  [labelText]="'Small'" 
  (radioChanged)="onRadioChange($event)">
</cometchat-radio-button>
```

### Radio Button Group

```html expandable
<div class="radio-group">
  <cometchat-radio-button
```

```typescript expandable
import { Component } from '@angular/core';
import { CometChatRadioButtonComponent } from '@cometchat/chat-uikit-angular';

@Component({
  selector: 'app-notification-settings',
  standalone: true,
  imports: [CometChatRadioButtonComponent],
  template: `
    <div class="settings-form">
      <h3>Notification Frequency</h3>
      
      <div class="radio-group">
        <cometchat-radio-button 
          [name]="'frequency'"
          [value]="'realtime'"
          [checked]="selectedFrequency === 'realtime'"
          [labelText]="'Real-time'" 
          (radioChanged)="onFrequencyChange($event)">
        </cometchat-radio-button>
        
        <cometchat-radio-button 
          [name]="'frequency'"
          [value]="'hourly'"
          [checked]="selectedFrequency === 'hourly'"
          [labelText]="'Hourly digest'" 
          (radioChanged)="onFrequencyChange($event)">
        </cometchat-radio-button>
        
        <cometchat-radio-button 
          [name]="'frequency'"
          [value]="'daily'"
          [checked]="selectedFrequency === 'daily'"
          [labelText]="'Daily digest'" 
          (radioChanged)="onFrequencyChange($event)">
        </cometchat-radio-button>
        
        <cometchat-radio-button 
          [name]="'frequency'"
          [value]="'never'"
          [checked]="selectedFrequency === 'never'"
          [labelText]="'Never'" 
          (radioChanged)="onFrequencyChange($event)">
        </cometchat-radio-button>
      </div>

      <h3>Message Sound</h3>
      
      <div class="radio-group">
        <cometchat-radio-button 
          [name]="'sound'"
          [value]="'default'"
          [checked]="selectedSound === 'default'"
          [labelText]="'Default'" 
          (radioChanged)="onSoundChange($event)">
        </cometchat-radio-button>
        
        <cometchat-radio-button 
          [name]="'sound'"
          [value]="'chime'"
          [checked]="selectedSound === 'chime'"
          [labelText]="'Chime'" 
          (radioChanged)="onSoundChange($event)">
        </cometchat-radio-button>
        
        <cometchat-radio-button 
          [name]="'sound'"
          [value]="'none'"
          [checked]="selectedSound === 'none'"
          [labelText]="'None'" 
          (radioChanged)="onSoundChange($event)">
        </cometchat-radio-button>
      </div>
    </div>
  `,
  styles: [`
    .settings-form {
      display: flex;
      flex-direction: column;
      gap: 24px;
      padding: 20px;
    }
    
    h3 {
      margin: 0 0 12px 0;
      font-size: 16px;
      font-weight: 600;
    }
    
    .radio-group {
      display: flex;
      flex-direction: column;
      gap: 12px;
    }
  `]
})
export class NotificationSettingsComponent {
  selectedFrequency = 'realtime';
  selectedSound = 'default';

  onFrequencyChange(event: { checked: boolean; labelText: string; id: string }): void {
    if (event.checked) {
      // Extract value from the radio button that emitted the event
      const radioButtons = document.getElementsByName('frequency') as NodeListOf<HTMLInputElement>;
      radioButtons.forEach(radio => {
        if (radio.checked) {
          this.selectedFrequency = radio.value;
        }
      });
      console.log('Frequency changed to:', this.selectedFrequency);
    }
  }

  onSoundChange(event: { checked: boolean; labelText: string; id: string }): void {
    if (event.checked) {
      const radioButtons = document.getElementsByName('sound') as NodeListOf<HTMLInputElement>;
      radioButtons.forEach(radio => {
        if (radio.checked) {
          this.selectedSound = radio.value;
        }
      });
      console.log('Sound changed to:', this.selectedSound);
    }
  }
}
```

## Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `checked` | `boolean` | `false` | Checked state of the radio button |
| `name` | `string` | `'radio-group'` | Name for grouping radio buttons. All radio buttons with the same name belong to the same group |
| `labelText` | `string` | `''` | Label text displayed next to the radio button |
| `disabled` | `boolean` | `false` | Disables the radio button, preventing interaction |
| `id` | `string` | `auto-generated` | Unique identifier for the radio button. Auto-generated if not provided |
| `value` | `any` | `''` | Value of the radio button. Used for form integration and identification |
| `ariaLabel` | `string` | `undefined` | Custom aria-label for accessibility (overrides default label for screen readers) |

## Events

| Event | Payload Type | Description |
|-------|-------------|-------------|
| `radioChanged` | `EventEmitter<{ checked: boolean; labelText: string; id: string }>` | Emitted when radio button state changes. Provides the checked state, label text, and unique id |

### Disabled State

When `disabled` is `true`:
- Radio button shows reduced opacity (0.5)
- Cursor changes to `not-allowed`
- Click and keyboard events are blocked
- ARIA attribute `aria-disabled="true"` is set
- Radio button is excluded from keyboard navigation

## Angular Forms Integration

The radio button implements `ControlValueAccessor`, allowing seamless integration with Angular forms:

### Reactive Forms

```typescript expandable

          [name]="'plan'"
          [value]="'free'"

        </cometchat-radio-button>
        
        <cometchat-radio-button 
          formControlName="plan"
          [name]="'plan'"
          [value]="'pro'"
          [labelText]="'Pro - $9/month'">
        </cometchat-radio-button>
        
        <cometchat-radio-button 
          formControlName="plan"
          [name]="'plan'"
          [value]="'enterprise'"
          [labelText]="'Enterprise - Contact us'">

      </div>
```

## Keyboard Accessibility

The radio button component provides full keyboard support:

| Key | Action |
|-----|--------|
| `Tab` | Move focus to the radio button group. Focus moves to the checked radio button, or the first radio button if none are checked |
| `Space` | Select the focused radio button |
| `Arrow Down` / `Arrow Right` | Move focus to and select the next radio button in the group. Wraps to the first radio button if at the end |
| `Arrow Up` / `Arrow Left` | Move focus to and select the previous radio button in the group. Wraps to the last radio button if at the beginning |

## Accessibility

The component follows WAI-ARIA best practices for radio buttons:

### ARIA Attributes

- `role="radio"`: Identifies the element as a radio button
- `aria-checked`: Indicates the checked state (`true` or `false`)
- `aria-disabled`: Indicates the disabled state when applicable

### Screen Reader Support

- Radio button label is properly associated with the input
- State changes are announced to screen readers
- Disabled state is communicated to assistive technologies

### Focus Management

- Visible focus indicators with 2px outline
- Focus outline uses primary color from CSS variables
- Focus is properly managed during keyboard navigation
- Disabled radio buttons are excluded from tab order

## Best Practices

### Radio Button Groups

1. **Always use the same `name`** for radio buttons that belong to the same group
2. **Provide meaningful labels** that clearly describe each option
3. **Set one radio button as checked by default** to avoid an empty selection state
4. **Use radio buttons for 2-7 options**. For more options, consider a dropdown

### Accessibility

1. **Wrap radio groups in a container** with `role="radiogroup"` and `aria-label`
2. **Provide clear, descriptive labels** for each radio button
3. **Don't rely solely on color** to indicate state
4. **Test with keyboard navigation** to ensure all options are accessible

### Forms Integration

1. **Use unique `value` attributes** for each radio button in a group
2. **Validate required selections** in your form logic
3. **Provide feedback** when a selection is required but not made
4. **Consider default selections** to improve user experience

## Integration and Isolation

### Event Listener Scoping

This component uses component-scoped event listeners that don't interfere with your application:

```typescript
@HostListener('keydown', ['$event'])
handleKeydown(event: KeyboardEvent): void {
  // Only handles events within this component
  // Arrow keys navigate between radio buttons in the same group
}
```

### Focus Management

Focus management is isolated to this component:
- Native radio element handles focus automatically
- Roving tabindex implemented for keyboard navigation
- Focus indicators only appear during keyboard navigation (`:focus-visible`)
- No global focus manipulation

### Radio Group Management

**Note**: The component uses `document.getElementsByName()` for radio group management:
- This is the standard HTML pattern for radio button groups
- Radio buttons are inherently grouped by the `name` attribute
- This approach is acceptable and follows W3C standards
- No accessibility impact - works correctly with screen readers

### Non-Interference

The component is designed to not interfere with your application:
- ✅ No global event listeners (except standard radio group queries)
- ✅ No document-level DOM manipulation
- ✅ Component-scoped styles with ViewEncapsulation
- ✅ Proper cleanup in `ngOnDestroy()`

### Multiple Instances

Multiple radio button groups work independently without interference:

```typescript
// Group 1 - works independently
<cometchat-radio-button [name]="'group1'" [value]="'a'"></cometchat-radio-button>
<cometchat-radio-button [name]="'group1'" [value]="'b'"></cometchat-radio-button>
```

### Cleanup

The component properly cleans up all resources when destroyed:
- Event listeners are removed automatically
- No subscriptions to clean up
- No memory leaks
