## Overview

The `Incoming Call` is a Component that serves as a visual representation when the user receives an incoming call, such as a voice call or video call, providing options to answer or decline the call.

Import path: `import { CometChatIncomingCall } from "@cometchat/chat-uikit-react";`

<Frame>
  <img src="/images/e63dbd67-incoming_calls_overview_web_screens-d0434dd2b66b78fe0bff43bdd34435d3.png" />
</Frame>

The `Incoming Call` is comprised of the following base components:

| Components          | Description                                                                                                                                    |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| cometchat-list-item | This component's view consists of avatar, status indicator , title, and subtitle. The fields are then mapped with the SDK's user, group class. |
| cometchat-button    | This component represents a button with optional icon and text.                                                                                |
| cometchat-avatar    | This component component displays an image or user's avatar with fallback to the first two letters of the username.                            |

<Note>
**Before using this component:** Ensure `CometChatUIKit.init(UIKitSettings)` has completed and the user is logged in via `CometChatUIKit.login("UID")`. See [React.js Integration](/ui-kit/react/react-js-integration).
</Note>

## Why Use / When to Use

- Display an incoming call notification when a voice or video call is received in real time
- Provide accept and decline controls so users can respond to incoming calls without leaving the chat interface
- Customize the call acceptance experience using `CallSettingsBuilder` to control audio-only vs. video call settings
- Override default accept and decline behavior with custom `onAccept` and `onDecline` callbacks for app-specific workflows
- Listen for call lifecycle events (`ccCallRejected`, `ccCallAccepted`, `ccCallEnded`) to trigger UI updates or analytics
- Replace default views (subtitle, leading, trailing, title, item) with custom JSX for a branded call experience
- Disable or customize the incoming call ringtone using `disableSoundForCalls` and `customSoundForCalls` props

### Filters

You can set `CallSettingsBuilder` in the Incoming Call Component to customise the calling experience. To know more about the filters available please refer to [CallSettingsBuilder](/sdk/javascript/direct-call#settings).

<Tabs>
<Tab title="TypeScript">
```ts lines
import { CometChatIncomingCall } from "@cometchat/chat-uikit-react";
import React from "react";

const IncomingCallsDemo = () => {
  const getCallSettingsBuilder = (call: CometChat.Call) => {
    return new CometChatCalls.CallSettingsBuilder()
      .setIsAudioOnlyCall(
        call?.getType() === CometChatUIKitConstants.MessageTypes.audio
          ? true
          : false
      )
      .build();
  };

  return <CometChatIncomingCall callSettingsBuilder={getCallSettingsBuilder} />;
};

export default IncomingCallsDemo;
```

### Events

[Events](/ui-kit/react/components-overview#events) are emitted by a `Component`. By using event you can extend existing functionality. Being global events, they can be applied in Multiple Locations and are capable of being Added or Removed.

The list of events emitted by the Incoming Call component is as follows.

| Event              | Description                                                                  |
| ------------------ | ---------------------------------------------------------------------------- |
| **ccCallRejected** | This event is triggered when the initiated call is rejected by the receiver. |
| **ccCallAccepted** | This event is triggered when the initiated call is accepted by the receiver. |
| **ccCallEnded**    | This event is triggered when the initiated call successfully ends.           |

<Tabs>
<Tab title="Add Listener">
```tsx lines

  (call: CometChat.Call) => {
    //Your Code
  }
);

  (call: CometChat.Call) => {
    //Your Code
  }
);

const ccCallEnded = CometChatCallEvents.ccCallEnded.subscribe(
  (call: CometChat.Call) => {
    //Your Code
  }
);
```

## Pagination, Batching, and Performance

Pagination does not apply to the Incoming Call component. The Incoming Call component displays a single incoming call notification and does not fetch or paginate lists of data. For list-based pagination, see the [Conversations](/ui-kit/react/conversations), [Users](/ui-kit/react/users), or [Call Logs](/ui-kit/react/call-logs) components.

## Error Handling and Retries

The Incoming Call component supports an `onError` callback prop that fires when an error occurs within the component. You can use this to log errors or display custom error UI.

See the `onError` callback example in [Basic Usage > Actions](#actions) for the implementation pattern. The `onError` prop receives a `CometChat.CometChatException` object containing the error details.

## Configuration and Environment

The CometChatIncomingCall component requires `CometChatUIKit.init()` to be called with a valid `appId` and `region` before rendering. The user must be logged in via `CometChatUIKit.login()`. The component depends on the `@cometchat/chat-uikit-react` package, which includes `@cometchat/chat-sdk-javascript` as a peer dependency.

The Incoming Call component also requires the CometChat Calls SDK (`@cometchat/calls-sdk-javascript`) as a peer dependency for call functionality. The `callSettingsBuilder` prop uses `CometChatCalls.CallSettingsBuilder` from this package to configure call settings.

React 18+ is supported. For Next.js, wrap the component in `useEffect` or use dynamic imports with `ssr: false` to avoid SSR hydration errors. See [Next.js Integration](/ui-kit/react/next-js-integration) and [React.js Integration](/ui-kit/react/react-js-integration) for setup details.

## Files and Media

Files and media handling does not apply to the Incoming Call component. The Incoming Call component displays call notifications with accept/decline controls but does not render or send media. For file and media support, see the [Message Composer](/ui-kit/react/message-composer) and [Message List](/ui-kit/react/message-list) components.

## Logging, Tracing, and Observability

To troubleshoot issues with the Incoming Call component (such as calls not being detected or callbacks not firing), enable CometChat SDK debug logging. Set the log level to `DEBUG` during `CometChat.init()` to see detailed SDK activity in the browser console. The component emits `ccCallRejected`, `ccCallAccepted`, and `ccCallEnded` events via `CometChatCallEvents` that can be used for observability and analytics. Refer to the CometChat SDK documentation for the full logging configuration API.
