## Overview

The CometChatGroups component renders a searchable, scrollable list of all available groups in your application. Each list item displays the group icon, name, member count as a subtitle, and a group type indicator (public, private, or password-protected). It supports selection modes, custom view slots, filtering via `GroupsRequestBuilder`, and CSS styling for building group directory and discovery experiences.

Import path: `import { CometChatGroups } from "@cometchat/chat-uikit-react";`

The Groups component is composed of the following BaseComponents:

| Components        | Description                                                                                                      |
| ----------------- | ---------------------------------------------------------------------------------------------------------------- |
| CometChatList     | A reusable container component having title, search box, customisable background and a list view.                |
| CometChatListItem | A component that renders data obtained from a Group object on a Tile having a title, subtitle, leading and trailing view. |

## Why Use / When to Use

- Display a searchable directory of all available groups in your application
- Allow users to select a group and navigate to the group chat view with `onItemClick`
- Filter groups by joined status, tags, or search keywords using `GroupsRequestBuilder`
- Show group type indicators (public, private, password-protected) for each group in the list
- Customize the appearance of each group item using view slots like `itemView`, `subtitleView`, and `leadingView`
- Enable multi-select mode for batch operations such as joining multiple groups or managing group memberships
- Display member counts as subtitles to provide context about group size and activity level

## Usage

### Actions

[Actions](/ui-kit/react/components-overview#actions) dictate how a component functions. They are divided into two types: Predefined and User-defined. You can override either type, allowing you to tailor the behavior of the component to fit your specific needs.

#### 1. onItemClick

The `onItemClick` event is activated when you click on the Group List item. This action does not come with any predefined behavior. You can override this event using the following code snippet.

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatGroups } from "@cometchat/chat-uikit-react";

const handleOnItemClick = (group: CometChat.Group) => {
  console.log("Selected group:", group);
  // Your custom action here
};

<CometChatGroups onItemClick={handleOnItemClick} />;
```

#### 2. onSelect

The `onSelect` action is activated when you select the done icon while in selection mode. This returns the `group` object along with the boolean flag `selected` to indicate if the group was selected or unselected.

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import {
  CometChatGroups,
  SelectionMode,
} from "@cometchat/chat-uikit-react";

const handleOnSelect = (group: CometChat.Group, selected: boolean) => {
  console.log("Group selected:", group.getName(), selected);
  // Your custom action on select
};

<CometChatGroups
  selectionMode={SelectionMode.multiple}
  onSelect={handleOnSelect}
/>;
```

#### 3. onError

This action listens for any errors that occur in the Groups component.

```tsx lines
import { CometChatGroups } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

const handleOnError = (error: CometChat.CometChatException) => {
  console.error("Groups error:", error);
  // Your exception handling code
};

<CometChatGroups onError={handleOnError} />;
```

### Filters

You can set `GroupsRequestBuilder` in the Groups component to filter the group list. You can modify the builder as per your specific requirements with multiple options available to know more refer to [GroupsRequestBuilder](/sdk/javascript/retrieve-groups).

The [GroupsRequestBuilder](/sdk/javascript/retrieve-groups) enables you to filter and customize the group list based on available parameters:

| Methods | Type | Description |
| --- | --- | --- |
| **setLimit** | number | Sets the number of groups that can be fetched in a single request, suitable for pagination |
| **setSearchKeyword** | String | Used for fetching groups matching the passed string |
| **joinedOnly** | boolean | To fetch only joined groups |
| **setTags** | Array\<String> | Used for fetching groups containing the passed tags |
| **withTags** | boolean | Used to fetch tags data along with the list of groups |

Filter by joined groups with a limit of two:

```tsx lines
import { CometChatGroups } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

<CometChatGroups
  groupsRequestBuilder={new CometChat.GroupsRequestBuilder()
    .setLimit(2)
    .joinedOnly(true)}
/>;
```

The `SearchRequestBuilder` uses [GroupsRequestBuilder](/sdk/javascript/retrieve-groups) to filter and customize the search list, keeping uniformity between the displayed Groups List and searched Group List:

```tsx lines
import { CometChatGroups } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

<CometChatGroups
  searchRequestBuilder={new CometChat.GroupsRequestBuilder()
    .setLimit(2)
    .setSearchKeyword("your keyword")}
/>;
```

[Events](/ui-kit/react/components-overview#events) are emitted by a `Component`. By using event you can extend existing functionality. Being global events, they can be applied in Multiple Locations and are capable of being Added or Removed.

The `Groups` component does not produce any events directly. However, you can subscribe to group lifecycle events via `CometChatGroupEvents` for custom handling across your application. ### Real-Time Group Events

The CometChatGroups component can respond to group lifecycle events via `CometChatGroupEvents`. You can subscribe to these events to react to group changes across your application.

Subscribe to group events:

```tsx lines
import { CometChatGroupEvents } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

// Subscribe to group created event
const groupCreatedSubscription = CometChatGroupEvents.ccGroupCreated.subscribe(
  (group: CometChat.Group) => {
    console.log("Group created:", group.getName());
  }
);

// Subscribe to group deleted event
const groupDeletedSubscription = CometChatGroupEvents.ccGroupDeleted.subscribe(
  (group: CometChat.Group) => {
    console.log("Group deleted:", group.getName());
  }
);

// Subscribe to group member added event
const memberAddedSubscription = CometChatGroupEvents.ccGroupMemberAdded.subscribe(
  (item: { userAddedIn: CometChat.Group; usersAdded: CometChat.User[]; addedBy: CometChat.User }) => {
    console.log("Members added to:", item.userAddedIn.getName());
  }
);

// Subscribe to group left event
const groupLeftSubscription = CometChatGroupEvents.ccGroupLeft.subscribe(
  (item: { leftGroup: CometChat.Group; leftUser: CometChat.User }) => {
    console.log("User left group:", item.leftGroup.getName());
  }
);
```

Unsubscribe from group events when no longer needed (e.g., in `useEffect` cleanup):

```tsx lines
groupCreatedSubscription?.unsubscribe();
groupDeletedSubscription?.unsubscribe();
memberAddedSubscription?.unsubscribe();
groupLeftSubscription?.unsubscribe();
```

| What you want to change | Where | Property/API | Example |
| --- | --- | --- | --- |
| Override behavior on user interaction | Component props | `on<Event>` callbacks | `onItemClick={(g) => setActive(g)}` |

| Customization Type | Prop Pattern | Example |
| --- | --- | --- |
| Callback actions | `on<Event>={(param) => {}}` | `onItemClick={(group) => {}}` |

| Toggle features | `hide<Feature>={true\|false}` | `hideSearch={true}` |
| Custom rendering | `<slot>View={(<params>) => JSX}` | `itemView={(group) => <div>...</div>}` |
| CSS overrides | Target `.cometchat-groups` class in global CSS | `.cometchat-groups .cometchat-list-item { ... }` |

## Pagination, Batching, and Performance

The CometChatGroups component fetches groups in pages using `GroupsRequestBuilder`. The default page size is 30 groups per request.

### Customizing Page Size

Use `setLimit` on the `GroupsRequestBuilder` to control how many groups are fetched per page:

```tsx lines
import { CometChatGroups } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

<CometChatGroups
  groupsRequestBuilder={
    new CometChat.GroupsRequestBuilder().setLimit(15)
  }
/>;
```

### Infinite Scroll

The component uses infinite scroll by default. As the user scrolls to the bottom of the list, the next page of groups is fetched automatically. No additional configuration is required.

### Custom Loading View

You can provide a custom loading indicator that displays while groups are being fetched:

```tsx lines
import { CometChatGroups } from "@cometchat/chat-uikit-react";

<CometChatGroups
  loadingView={<div className="custom-loader">Loading groups...</div>}
/>;
```

## Error Handling and Retries

The CometChatGroups component displays a default error view when group fetching fails. You can customize this behavior using the `onError`, `hideError`, and `errorView` props.

### Default Behavior

When an error occurs (such as a network failure or authentication issue), the component renders a built-in error state and logs the error to the console.

### onError Callback

Use the `onError` prop to intercept errors and implement custom error handling logic:

```tsx lines
import { CometChatGroups } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

const handleOnError = (error: CometChat.CometChatException) => {
  console.error("Groups error:", error.message);
  // Send to your error tracking service
};

<CometChatGroups onError={handleOnError} />;
```

### Hiding the Error View

Set `hideError` to `true` to suppress the default error UI entirely:

```tsx lines
import { CometChatGroups } from "@cometchat/chat-uikit-react";

<CometChatGroups hideError={true} />;
```

### Custom Error View

Provide a custom error component using the `errorView` prop:

```tsx lines
import { CometChatGroups } from "@cometchat/chat-uikit-react";

<CometChatGroups
  errorView={
    <div className="custom-error">
      <p>Something went wrong. Please try again.</p>
    </div>
  }
/>;
```

## Configuration and Environment

### CometChat Initialization

The CometChatGroups component requires `CometChatUIKit.init()` to be called before rendering. The initialization requires `appId`, `region`, and `authKey`:

```tsx lines
import { CometChatUIKit, UIKitSettingsBuilder } from "@cometchat/chat-uikit-react";

const settings = new UIKitSettingsBuilder()
  .setAppId("APP_ID")
  .setRegion("REGION")
  .setAuthKey("AUTH_KEY")
  .build();

await CometChatUIKit.init(settings);
await CometChatUIKit.login("USER_UID");
```

### Required Packages

The Groups component requires only the core UI Kit package:

- `@cometchat/chat-uikit-react` — the React UI Kit containing all components
- `@cometchat/chat-sdk-javascript` — installed automatically as a peer dependency

### Supported React Versions and Frameworks

- React 18+ is supported
- Compatible with Next.js (wrap in `useEffect` or use dynamic imports with `ssr: false` to avoid SSR hydration errors), React Router, and Astro
- For Next.js integration details, see [Next.js Integration](/ui-kit/react/next-js-integration)

## Files and Media

Files and media handling does not apply to the Groups component. The group list displays group metadata but does not render or send media directly. For file and media support, see the [Message Composer](/ui-kit/react/message-composer) and [Message List](/ui-kit/react/message-list) components.

## Logging, Tracing, and Observability

### Enabling Debug Logging

Enable CometChat SDK debug logging to troubleshoot group fetching and real-time event issues:

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";

const appSettings = new CometChat.AppSettingsBuilder()
  .subscribePresenceForAllUsers()
  .setRegion("REGION")
  .autoEstablishSocketConnection(true)
  .build();

CometChat.init("APP_ID", appSettings).then(() => {
  CometChat.setLogLevel(CometChat.LogLevel.DEBUG);
});
```

### Observable Events

The component can respond to group lifecycle events through `CometChatGroupEvents` that can be used for observability:

- `ccGroupCreated` — fired when a new group is created
- `ccGroupDeleted` — fired when a group is deleted
- `ccGroupMemberAdded` — fired when members are added to a group
- `ccGroupLeft` — fired when a user leaves a group

You can subscribe to these events to log user actions or feed analytics dashboards. See the [Real-Time Group Events](#real-time-group-events) subsection in Advanced Features for subscribe/unsubscribe patterns.
