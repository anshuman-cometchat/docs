Customize the CometChat UI Kit appearance using CSS variables for colors, fonts, and dark mode. This page is for developers who want predictable theming with clear override rules.

## When to use this

- You want to match the UI Kit to your brand colors and typography.
- You need light and dark mode palettes.
- You want per-component styling overrides.
- You need deterministic override precedence across components.
- You want to override styles without editing UI Kit source code.

## Prerequisites

- CometChat React UI Kit v6 installed: `@cometchat/chat-uikit-react`.
- Base stylesheet imported in `App.css`: `@import url("@cometchat/chat-uikit-react/css-variables.css");`
- `App.css` imported in your app entry (for example `src/App.tsx`).
- UI Kit rendered under a `.cometchat` root wrapper.
- Use global CSS (not CSS Modules) for overrides.

## Quick start

1. Add the base UI Kit variables to `src/App.css`.

```css
@import url("@cometchat/chat-uikit-react/css-variables.css");
```

What this does: Loads the default UI Kit tokens so your overrides can apply.

2. Import the stylesheet in `src/App.tsx`.

```tsx
import "./App.css";

export default function App() {
  return <div className="cometchat-root">{/* UI Kit components */}</div>;
}
```

What this does: Ensures your global CSS is applied.

3. Add a global theme override in `src/App.css`.

```css
@import url("@cometchat/chat-uikit-react/css-variables.css");

.cometchat {
  --cometchat-primary-color: #f76808;
  --cometchat-font-family: "Inter", sans-serif;
}
```

What this does: Changes the primary accent color and the global font.

4. (Optional) Add a component-level override.

```css
@import url("@cometchat/chat-uikit-react/css-variables.css");

.cometchat .cometchat-conversations {
  --cometchat-primary-color: #e63946;
}
```

What this does: Overrides only the Conversations component’s primary color.

5. (Optional) Add a dark mode override.

```css
@import url("@cometchat/chat-uikit-react/css-variables.css");

.cometchat-root[data-theme="dark"] .cometchat {
  --cometchat-primary-color: #f76808;
}
```

What this does: Applies an alternate palette when `data-theme="dark"` is set on the wrapper.

## Core concepts

<Info>
**Quick Reference for AI Agents & Developers**

```css
@import url("@cometchat/chat-uikit-react/css-variables.css");

.cometchat {
  --cometchat-primary-color: #f76808;
  --cometchat-font-family: "Inter", sans-serif;
}

.cometchat .cometchat-conversations {
  --cometchat-primary-color: #e63946;
}

.cometchat-root[data-theme="dark"] .cometchat {
  --cometchat-primary-color: #f76808;
}
```
</Info>

- Root wrapper: UI Kit components render under `.cometchat` and should be styled through that scope.
- Token overrides: Use CSS variables on `.cometchat` for global changes.
- Component overrides: Use `.cometchat .cometchat-<component>` to scope changes to a single component.
- Precedence: Element-level CSS overrides > component-level variables > global variables > default tokens.
- Dark mode: Use either `data-theme` or `@media (prefers-color-scheme: dark)`.

### Top tokens (quick mapping)

| Token | Common usage (varies by component) |
| --- | --- |
| `--cometchat-primary-color` | Primary accent color (buttons, outgoing bubbles, active states) |
| `--cometchat-extended-primary-color-900` | Darker primary shade (outgoing bubble shade) |
| `--cometchat-extended-primary-color-500` | Mid primary shade (secondary accents/hover) |
| `--cometchat-neutral-color-300` | Neutral surface (incoming bubbles/panels) |
| `--cometchat-background-color-03` | Panel background surface |
| `--cometchat-text-color-highlight` | Highlight text color |
| `--cometchat-icon-color-highlight` | Highlight icon color |
| `--cometchat-message-seen-color` | Seen/read indicator color |
| `--cometchat-font-family` | Global font family |
| `--cometchat-radius-max` | Maximum corner radius used across UI elements |

### Dark and light theme support

What you’re changing: Provide dark mode overrides.
File: `src/App.tsx` and `src/App.css`
Applies to: `.cometchat-root[data-theme="dark"] .cometchat`
Default behavior: Light mode tokens apply.
Override: Set `data-theme` and add dark overrides in CSS.
Verify: Theme switches based on `data-theme` value.

<Frame>
  <img src="/images/3a7b0c73-theming_dark_mode_default-fff438886065da229f12cfb3b8904f27.png" />
</Frame>

```tsx
import { useEffect, useState } from "react";
import "./App.css";

export default function App() {
  const [theme, setTheme] = useState("light");

  useEffect(() => {
    const mediaQuery = window.matchMedia("(prefers-color-scheme: dark)");
    setTheme(mediaQuery.matches ? "dark" : "light");

    const handleThemeChange = (e: MediaQueryListEvent) => {
      setTheme(e.matches ? "dark" : "light");
    };

    mediaQuery.addEventListener("change", handleThemeChange);
    return () => mediaQuery.removeEventListener("change", handleThemeChange);
  }, []);

  return (
    <div className="cometchat-root" data-theme={theme}>
      {/* UI Kit components */}
    </div>
  );
}
```

What this does: Sets `data-theme` based on OS preference.

## Common pitfalls and fixes

| Symptom | Cause | Fix |
| --- | --- | --- |
| CSS has no effect | CSS file not imported in app entry | Add `import "./App.css";` in `App.tsx`. |
| CSS has no effect | Base stylesheet not imported | Add `@import url("@cometchat/chat-uikit-react/css-variables.css");` at top of `App.css`. |
| Overrides not applying | Missing `.cometchat` scope in selector | Ensure overrides are scoped under `.cometchat`. |
| Selectors do not match | Using CSS Modules (hashed class names) | Move rules to a global stylesheet, not `*.module.css`. |
| Component override not working | Missing `.cometchat` parent in selector | Use `.cometchat .cometchat-conversations`, not just `.cometchat-conversations`. |
| Dark mode unchanged | `data-theme` missing or mismatch | Set `data-theme="dark"` on the wrapper and target `.cometchat-root[data-theme="dark"] .cometchat`. |
| Font not changing | `--cometchat-font-family` set on wrong element | Set it on `.cometchat` or the `.cometchat` wrapper. |
| Styles leak into host app | Missing `.cometchat` scope prefix | Always scope overrides under `.cometchat`. |
| Token change has no visible effect | Token does not control that property | Use [Color Resources](/ui-kit/react/theme/color-resources) to confirm token usage. |

## FAQ

**Do I need to use CSS variables?**
Yes. The UI Kit theming system is designed around CSS variables.

**Where should I place my overrides?**
In a global stylesheet like `App.css`, scoped under `.cometchat`.

**Can I use CSS Modules?**
No. CSS Modules generate hashed class names that do not match UI Kit selectors.
