## Overview

CometChatThreadHeader is a Component that displays the parent message & number of replies of thread.

Import path: `import { CometChatThreadHeader } from "@cometchat/chat-uikit-react";`

<Frame>
  <img src="/images/a5456902-threaded_message_preview_overview_web_screens-0f6df1fc848c99494f5ef81c5c44393f.png" />
</Frame>

<Note>
**Before using this component:** Ensure `CometChatUIKit.init(UIKitSettings)` has completed and the user is logged in via `CometChatUIKit.login("UID")`. See [React.js Integration](/ui-kit/react/react-js-integration).
</Note>

## Why Use / When to Use

- Display the parent message of a threaded conversation at the top of the thread view
- Show the reply count for a thread so users can gauge discussion activity
- Provide a close button so users can exit the thread and return to the main conversation
- Customize the message bubble rendering within the thread header using custom views
- Control visibility of date headers and reply counts via boolean props
- Customize date and time formatting for separator and message timestamps using `CalendarObject`

## Usage

### Integration

The following code snippet illustrates how you can directly incorporate the CometChatThreadHeader component into your Application.

<Tabs>
<Tab title="TypeScript">
```tsx lines
import React from "react";
import { CometChatThreadHeader } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

export function ThreadHeaderDemo() {
  const [parentMessage, setParentMessage] =
    React.useState<CometChat.BaseMessage>();
  const [chatWithUser, setChatWithUser] = React.useState<CometChat.User>();

  React.useEffect(() => {
    CometChat.getUser("uid").then((user) => {
      setChatWithUser(user);
    });
    CometChat.getMessageDetails("Parent Message Id").then((message) => {
      setParentMessage(message);
    });
  }, []);

  return chatWithUser && parentMessage ? (
    <CometChatThreadHeader parentMessage={parentMessage} />
  ) : null;
}
```

For advanced-level customization, you can set custom views to the component. This lets you tailor each aspect of the component to fit your exact needs and application aesthetics. You can create and define your views, layouts, and UI elements and then incorporate those into the component.

### Separator DateTime Format

The `separatorDateTimeFormat` property allows you to customize the **Date Separator** timestamp displayed in the Threaded Message Preview.

Default Date Time Format:

```ruby
new CalendarObject({
    today: `DD MMM, YYYY`,   // Example: "25 Jan, 2025"
    yesterday: `DD MMM, YYYY`, // Example: "25 Jan, 2025"
    otherDays: `DD MMM, YYYY`,  // Example: "25 Jan, 2025"
});
```

The following example demonstrates how to modify the **Date Separator** timestamp format using a custom [`CalendarObject`](/ui-kit/react/localize#calendarobject).

<Tabs>
<Tab title="TypeScript">
```ts lines
import {
  CometChatThreadHeader,
  CalendarObject
} from "@cometchat/chat-uikit-react";

// Define a custom date format pattern
function getDateFormat() {
  const dateFormat = new CalendarObject({
    today: `hh:mm A`, // Example: "10:30 AM"
    yesterday: `[yesterday]`, // Example: "yesterday"
    otherDays: `DD/MM/YYYY`, // Example: "25/05/2025"
  });
  return dateFormat;
}

// Apply the custom format to the CometChatThreadHeader component
<CometChatThreadHeader separatorDateTimeFormat={getDateFormat()} />;
```

</Tab>

</Tabs>

<Note>

**Fallback Mechanism**

* If you **do not** pass any property in the [**CalendarObject**](/ui-kit/react/localize#calendarobject), the component will first check the [**global configuration**](/ui-kit/react/localize#customisation). If the property is **also missing in the global configuration**, it will **fallback to the component's default formatting**.

</Note>

### Message SentAt DateTime Format

## Pagination, Batching, and Performance

Pagination does not apply to the ThreadHeader component. The ThreadHeader displays the parent message and reply count for a single thread and does not fetch or paginate lists of data. For list-based pagination, see the [Conversations](/ui-kit/react/conversations), [Users](/ui-kit/react/users), or [Message List](/ui-kit/react/message-list) components.

## Error Handling and Retries

The ThreadHeader component supports an `onError` callback prop that fires when an error occurs during rendering or data fetching. You can use this to log errors or display custom error UI. The `onError` prop receives a `CometChat.CometChatException` object containing the error details — see the Functionality table above for the inline usage pattern: `onError={(error: CometChat.CometChatException) => console.log(error)}`.

## Configuration and Environment

The CometChatThreadHeader component requires `CometChatUIKit.init()` to be called with a valid `appId` and `region` before rendering. The user must be logged in via `CometChatUIKit.login()`. The component depends on the `@cometchat/chat-uikit-react` package, which includes `@cometchat/chat-sdk-javascript` as a peer dependency.

React 18+ is supported. For Next.js, wrap the component in `useEffect` or use dynamic imports with `ssr: false` to avoid SSR hydration errors. See [Next.js Integration](/ui-kit/react/next-js-integration) and [React.js Integration](/ui-kit/react/react-js-integration) for setup details.

## Files and Media

Files and media handling does not apply to the ThreadHeader component. The header displays the parent message and reply count but does not render or send media directly. For file and media support, see the [Message Composer](/ui-kit/react/message-composer) and [Message List](/ui-kit/react/message-list) components.

## Logging, Tracing, and Observability

To troubleshoot issues with the ThreadHeader component (such as the parent message not displaying or reply count not updating), enable CometChat SDK debug logging. Set the log level to `DEBUG` during `CometChat.init()` to see detailed SDK activity in the browser console. Refer to the CometChat SDK documentation for the full logging configuration API.
