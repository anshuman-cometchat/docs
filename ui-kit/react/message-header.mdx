## Overview

`MessageHeader` is a Component that showcases the [User](/sdk/javascript/users-overview) or [Group](/sdk/javascript/groups-overview) details in the toolbar. Furthermore, it also presents a typing indicator and a back navigation button for ease of use.

Import path: `import { CometChatMessageHeader } from "@cometchat/chat-uikit-react";`

<Frame>
  <img src="/images/af3844a0-messages_header_overview_web_screens-e2e210ec4afa809b4d4dd038df2b4073.png" />
</Frame>

<Note>
**Before using this component:** Ensure `CometChatUIKit.init(UIKitSettings)` has completed and the user is logged in via `CometChatUIKit.login("UID")`. See [React.js Integration](/ui-kit/react/react-js-integration).
</Note>

The `MessageHeader` is comprised of the following components:

| Component         | Description                                                                                                                                    |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| CometChatListItem | This component's view consists of avatar, status indicator , title, and subtitle. The fields are then mapped with the SDK's user, group class. |
| Back Button       | BackButton that allows users to navigate back from the current activity or screen to the previous one.                                         |

## Why Use / When to Use

- Display user or group details (avatar, name, status) at the top of a chat view
- Show a real-time typing indicator when the other participant is composing a message
- Provide back navigation so users can return to the conversation list from the message view
- Toggle visibility of voice and video call buttons for call-enabled conversations
- Customize the header layout with custom view slots for title, subtitle, leading, trailing, and auxiliary button areas
- Show or hide the user's online/offline status indicator based on your app's requirements
- Enable a conversation summary button for AI-powered message summarization

### Actions

[Actions](/ui-kit/react/components-overview#actions) dictate how a component functions. They are divided into two types: Predefined and User-defined. You can override either type, allowing you to tailor the behavior of the component to fit your specific needs.

#### 1. OnBack

**Example**

#### LastActiveAt Date Time Format

The `lastActiveAtDateTimeFormat` property allows you to customize the **last active** timestamp displayed in the message header.

Default Date Time Format:

```ruby lines
new CalendarObject({
  today: `[Last seen DD MMM at] hh:mm A`, // Example: "today at 10:30 AM"
  yesterday: `[Last seen DD MMM at] hh:mm A`, // Example: "yesterday at 08:15 PM"
  otherDays: `[Last seen DD MMM at] hh:mm A`, // Example: "25, Jan 05:45 PM"
  relativeTime: {
    hour: `Last seen  %d hour ago`, // Example: "1 hour ago"
    minute: `Last seen  %d minute ago`, // Example: "1 minute ago"
    minutes: `Last seen  %d minutes ago`, // Example: "5 minutes ago"
  },
});
```

The following example demonstrates how to modify the **last active** timestamp format using a custom [`CalendarObject`](/ui-kit/react/localize#calendarobject).

<Tabs>
<Tab title="TypeScript">
```ts lines
import {
  CometChatMessageHeader,
  CalendarObject,
} from "@cometchat/chat-uikit-react";

// Define a custom date format pattern
function getDateFormat() {
  const dateFormat = new CalendarObject({
    today: `hh:mm A`, // Example: "10:30 AM"
    yesterday: `[yesterday]`, // Example: "yesterday"
    otherDays: `DD MM YYYY`, // Example: "25 01 2025"
  });
  return dateFormat;
}

// Apply the custom format to the CometChatMessageHeader component
<CometChatMessageHeader
  group={groupObj}
  lastActiveAtDateTimeFormat={getDateFormat()}
/>;
```

</Tab>

</Tabs>

<Note>

**Fallback Mechanism**

* If you **do not** pass any property in the [**CalendarObject**](/ui-kit/react/localize#calendarobject), the component will first check the [**global configuration**](/ui-kit/react/localize#customisation). If the property is **also missing in the global configuration**, it will **fallback to the component's default formatting**.

</Note>

## Pagination, Batching, and Performance

Pagination does not apply to the MessageHeader component. The MessageHeader displays details for a single user or group conversation and does not fetch or paginate lists of data. For list-based pagination, see the [Conversations](/ui-kit/react/conversations), [Users](/ui-kit/react/users), or [Message List](/ui-kit/react/message-list) components.

## Error Handling and Retries

The MessageHeader component supports an `onError` callback prop that fires when an error occurs during rendering or data fetching. You can use this to log errors or display custom error UI.

See the `onError` callback example in [Basic Usage > Actions](#actions) for the implementation pattern. The `onError` prop receives a `CometChat.CometChatException` object containing the error details.

<AccordionGroup>
<Accordion title="Common Failure Modes">

| Symptom | Cause | Fix |
| --- | --- | --- |
| Component doesn't render | `CometChatUIKit.init()` not called or not awaited | Ensure init completes before rendering. See [Methods](/ui-kit/react/methods) |
| Component renders but shows no data | User not logged in | Call `CometChatUIKit.login("UID")` after init |
| Typing indicator not showing | No active typing in the conversation | Typing indicator appears only when the other user is actively typing |
| Back button not visible | `hideBackButton={true}` is set | Remove or set `hideBackButton={false}` |
| SSR hydration error | Component uses browser APIs on server | Wrap in `useEffect` or dynamic import with `ssr: false`. See [Next.js Integration](/ui-kit/react/next-js-integration) |

## Configuration and Environment

The CometChatMessageHeader component requires `CometChatUIKit.init()` to be called with a valid `appId` and `region` before rendering. The user must be logged in via `CometChatUIKit.login()`. The component depends on the `@cometchat/chat-uikit-react` package, which includes `@cometchat/chat-sdk-javascript` as a peer dependency.

React 18+ is supported. For Next.js, wrap the component in `useEffect` or use dynamic imports with `ssr: false` to avoid SSR hydration errors. See [Next.js Integration](/ui-kit/react/next-js-integration) and [React.js Integration](/ui-kit/react/react-js-integration) for setup details.

## Files and Media

Files and media handling does not apply to the MessageHeader component. The header displays user or group details but does not render or send media. For file and media support, see the [Message Composer](/ui-kit/react/message-composer) and [Message List](/ui-kit/react/message-list) components.

## Logging, Tracing, and Observability

To troubleshoot issues with the MessageHeader component (such as user status not updating or typing indicators not appearing), enable CometChat SDK debug logging. Set the log level to `DEBUG` during `CometChat.init()` to see detailed SDK activity in the browser console. Refer to the CometChat SDK documentation for the full logging configuration API.
