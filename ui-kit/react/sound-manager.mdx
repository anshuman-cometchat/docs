`CometChatSoundManager` is a helper class for managing and playing audio cues in the UI Kit — incoming/outgoing calls and messages.

---

## Methods

### play

Plays the default or custom audio resource for a given sound event.

| Signature | Description |
| --- | --- |
| `play(sound: Sound)` | Plays the predefined sound for the event |
| `play(sound: Sound, filePath: string)` | Plays a custom MP3 file for the event |

### pause

Pauses the currently playing sound.

| Signature | Description |
| --- | --- |
| `pause()` | Pauses any sound currently being played |

---

## Sound Events

| Event | When it plays |
| --- | --- |
| `Sound.incomingCall` | Incoming call detected |
| `Sound.outgoingCall` | Outgoing call initiated |
| `Sound.incomingMessage` | New message received |
| `Sound.outgoingMessage` | Message sent |

---

## Usage

```ts
import { CometChatSoundManager, Sound } from "@cometchat/chat-uikit-react";

// Play default incoming call sound
CometChatSoundManager.play(Sound.incomingCall);

// Play custom sound for incoming call
CometChatSoundManager.play(Sound.incomingCall, "MP3_FILE_ASSET_PATH");

// Pause the ongoing sound
CometChatSoundManager.pause();
```

---

## Common Failure Modes

| Symptom | Cause | Fix |
| --- | --- | --- |
| No sound plays | Browser autoplay policy blocks audio | User must interact with the page first before audio can play |
| Custom sound not playing | Invalid file path or unsupported format | Ensure the path is correct and the file is a valid MP3 |
| Sound keeps playing | `pause()` not called | Call `CometChatSoundManager.pause()` to stop playback |
