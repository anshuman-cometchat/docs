## Overview

`CometChatAIAssistantChat` is a composite component that assembles the message header, message list, and message composer to provide an AI agent chat experience. It supports streaming responses, quick starter suggestions, "New Chat" to reset context, and a chat history sidebar.

Import path: `import { CometChatAIAssistantChat } from "@cometchat/chat-uikit-react";`

<Frame>
  <img src="/images/react-uikit_ai-assistant-chat-overview.png" />
</Frame>

<Note>
**Before using this component:** Ensure `CometChatUIKit.init(UIKitSettings)` has completed and the user is logged in via `CometChatUIKit.login("UID")`. See [React.js Integration](/ui-kit/react/react-js-integration).
</Note>

## Why Use / When to Use

- Build a dedicated AI agent chat experience with streaming responses and quick starter suggestions
- Provide a "New Chat" button to reset conversation context so users can start fresh topics
- Display a chat history sidebar so users can revisit previous AI conversations
- Customize the empty state with greeting messages, intro text, and images to match your brand
- Hook into assistant tool/function calls during replies using `CometChatAIAssistantTools`
- Override header, message list, and composer behavior through view slot props for full control
- Handle errors from the underlying header, list, or composer via the `onError` callback

***

### Filters

The AI Assistant Chat component does not expose dedicated filter or `RequestBuilder` props. The component manages its own data fetching internally for AI conversations.

### Events

The AI Assistant Chat component does not emit global events directly. It composes the message header, message list, and message composer internally. For event handling, use the `onBackButtonClicked`, `onCloseButtonClicked`, `onSendButtonClick`, and `onError` callback props documented in the [Actions](#actions) section above.

Below is a list of customizations along with corresponding code snippets

| Property | Description | Code |
| --- | --- | --- |
| `user` | Required `CometChat.User` representing the AI agent. | `user={agent}` |
| `streamingSpeed` | Characters-per-second speed for streaming replies. Default `30`. | `streamingSpeed={50}` |
| `suggestedMessages` | Array of quick prompts for the empty state. | `suggestedMessages={["Help", "Summarize"]}` |
| `hideSuggestedMessages` | Hide the suggestions section. | `hideSuggestedMessages={true}` |
| `hideNewChat` | Hide the New Chat button in header. | `hideNewChat={true}` |
| `hideChatHistory` | Hide the History button/sidebar. | `hideChatHistory={true}` |
| `showBackButton` | Show back button in header. | `showBackButton` |
| `showCloseButton` | Show close button in header. | `showCloseButton` |
| `onBackButtonClicked` | Back button handler. | `onBackButtonClicked={() => {}}` |
| `onCloseButtonClicked` | Close button handler. | `onCloseButtonClicked={() => {}}` |
| `onSendButtonClick` | Send button handler. | `onSendButtonClick={() => {}}` |
| `onError` | Error handler. | `onError={handleError}` |
| `emptyView` | Custom empty state for the list. | `emptyView={<Empty/>}` |
| `loadingView` | Custom loading view. | `loadingView={<Loading/>}` |
| `errorView` | Custom error view. | `errorView={<Error/>}` |
| `emptyChatGreetingView` | Custom empty title (default uses metadata.greetingMessage or user name). | `emptyChatGreetingView={<h3/>}` |
| `emptyChatIntroMessageView` | Custom empty subtitle (default uses metadata.introductoryMessage). | `emptyChatIntroMessageView={<p/>}` |
| `emptyChatImageView` | Custom empty image. | `emptyChatImageView={<img/>}` |
| `aiAssistantTools` | Provide tool/function handlers for the assistant. | `aiAssistantTools={tools}` |
| `loadLastAgentConversation` | Loads the most recent existing agent conversation if one is available. | `loadLastAgentConversation={true}` |
| `parentMessageId` | The parent message ID to load a specific agent conversation. Takes priority over `loadLastAgentConversation`. | `parentMessageId={12345}` |

## Advanced Features

For advanced-level customization, you can set custom views to the component. This lets you tailor each aspect of the component to fit your exact needs and application aesthetics. You can create and define your views, layouts, and UI elements and then incorporate those into the component.

### Custom View Slots

#### Header Views

Customize header sections using the following props: `headerItemView`, `headerTitleView`, `headerSubtitleView`, `headerLeadingView`, `headerTrailingView`, and `headerAuxiliaryButtonView`. These customizations are done in the similar way as the [Message Header](/ui-kit/react/message-header#advanced-features) component.

<Note>
The header's "New Chat" and "History" buttons are built-in. You can override them by providing a custom `headerAuxiliaryButtonView`.
</Note>

#### Assistant Tools

Pass an instance of `CometChatAIAssistantTools` to enable tool/function calls during assistant replies.

<Tabs>
<Tab title="TypeScript">
```tsx lines
import React from "react";
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatAIAssistantChat } from "@cometchat/chat-uikit-react";
import { CometChatAIAssistantTools } from "@cometchat/chat-uikit-react";

export function AIAssistantChatDemo = () => {
    const [agent, setAgent] = React.useState<CometChat.User>();

    React.useEffect(() => {
        CometChat.getUser("assistant_uid").then((u) => setAgent(u));
    }, []);

    const tools = new CometChatAIAssistantTools({
        getCurrentWeather: ({ location }: { location: string }) => {
            // call your backend or a public API
            console.log("Fetching weather for", location);
        },
        createTicket: ({ title }: { title: string }) => {
            // open your internal ticketing flow
            console.log("Create ticket:", title);
        },
    });

    if (!agent) return null;

    return (
        <CometChatAIAssistantChat user={agent} aiAssistantTools={tools} />
    );
```

## Pagination, Batching, and Performance

Pagination does not apply to the AI Assistant Chat component. The component manages AI conversation data internally and does not expose `RequestBuilder` or `setLimit` props. For list-based pagination, see the [Conversations](/ui-kit/react/conversations), [Users](/ui-kit/react/users), or [Message List](/ui-kit/react/message-list) components.

## Error Handling and Retries

The AI Assistant Chat component supports an `onError` callback prop that fires when an error occurs within the underlying header, list, or composer. You can use this to log errors or display custom error UI.

See the `onError` callback example in [Basic Usage > Actions](#actions) for the implementation pattern. The `onError` prop receives a `CometChat.CometChatException` object containing the error details.

The component also supports `errorView` to render a custom error state, and `loadingView` to render a custom loading state. See the [Functionality](#functionality) table for these props.

## Configuration and Environment

The CometChatAIAssistantChat component requires `CometChatUIKit.init()` to be called with a valid `appId` and `region` before rendering. The user must be logged in via `CometChatUIKit.login()`. Additionally, a `CometChat.User` representing the AI agent must be fetched via `CometChat.getUser("assistant_uid")` and passed as the `user` prop.

The component depends on the `@cometchat/chat-uikit-react` package, which includes `@cometchat/chat-sdk-javascript` as a peer dependency. The AI agent must be configured in the CometChat Dashboard for streaming responses to work.

React 18+ is supported. For Next.js, wrap the component in `useEffect` or use dynamic imports with `ssr: false` to avoid SSR hydration errors. See [Next.js Integration](/ui-kit/react/next-js-integration) and [React.js Integration](/ui-kit/react/react-js-integration) for setup details.

## Files and Media

Files and media handling does not apply to the AI Assistant Chat component. The AI Assistant Chat component provides a text-based AI conversation experience and does not render or send media attachments. For file and media support, see the [Message Composer](/ui-kit/react/message-composer) and [Message List](/ui-kit/react/message-list) components.

## Logging, Tracing, and Observability

To troubleshoot issues with the AI Assistant Chat component (such as the agent not responding or streaming not working), enable CometChat SDK debug logging. Set the log level to `DEBUG` during `CometChat.init()` to see detailed SDK activity in the browser console. The `onError` callback prop can be used to capture and log errors from the underlying header, list, or composer for observability and analytics. Refer to the CometChat SDK documentation for the full logging configuration API.
