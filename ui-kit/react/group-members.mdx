## Overview

The CometChatGroupMembers component displays all users added or invited to a group, showing their roles, scopes, and online status. Members can communicate in real-time via messaging, voice and video calls, and other collaborative features. The component supports member management actions such as kicking, banning, and scope changes, along with customizable views and filtering via `GroupMembersRequestBuilder`.

Import path: `import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";`

## Why Use / When to Use

- Display a scrollable list of all members in a specific group with their roles (owner, admin, moderator, participant)
- Allow users to select a group member and trigger custom actions with `onItemClick`
- Filter group members by scope, search keyword, or custom criteria using `GroupMembersRequestBuilder`
- Enable member management actions such as kicking, banning, and changing member scopes directly from the list
- Customize the appearance of each member item using view slots like `itemView`, `subtitleView`, `leadingView`, and `trailingView`
- Enable multi-select mode for batch operations on group members
- Show online/offline status indicators for each group member in real time

## Usage

### Actions

[Actions](/ui-kit/react/components-overview#actions) dictate how a component functions. They are divided into two types: Predefined and User-defined. You can override either type, allowing you to tailor the behavior of the component to fit your specific needs.

#### 1. onItemClick

The `onItemClick` event is activated when you click on a group member list item. This action does not come with any predefined behavior. You can override this event to handle member selection.

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

const handleOnItemClick = (groupMember: CometChat.GroupMember) => {
  console.log("Selected member:", groupMember.getName());
  // Your custom action here
};

<CometChatGroupMembers group={chatGroup} onItemClick={handleOnItemClick} />;
```

#### 2. onSelect

The `onSelect` action is activated when you select the done icon while in selection mode. This returns the selected group member and the selection state.

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import {
  CometChatGroupMembers,
  SelectionMode,
} from "@cometchat/chat-uikit-react";

const handleOnSelect = (
  groupMember: CometChat.GroupMember,
  selected: boolean
) => {
  console.log("Member selected:", groupMember.getName(), selected);
  // Your custom selection action
};

<CometChatGroupMembers
  group={chatGroup}
  selectionMode={SelectionMode.multiple}
  onSelect={handleOnSelect}
/>;
```

#### 3. onEmpty

The `onEmpty` callback is executed when the group members list has no data to display.

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

const handleOnEmpty = () => {
  console.log("No group members found");
};

<CometChatGroupMembers group={chatGroup} onEmpty={handleOnEmpty} />;
```

#### 4. onError

The `onError` callback listens for any errors that occur in the Group Members component.

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

const handleOnError = (error: CometChat.CometChatException) => {
  console.error("Group members error:", error.message);
  // Your exception handling code
};

<CometChatGroupMembers group={chatGroup} onError={handleOnError} />;
```

### Filters

You can set `GroupMembersRequestBuilder` in the Group Members component to filter the member list. You can modify the builder as per your specific requirements with multiple options available to know more refer to [GroupMembersRequestBuilder](/sdk/javascript/retrieve-group-members).

The following parameters are available in [GroupMembersRequestBuilder](/sdk/javascript/retrieve-group-members):

| Methods | Type | Description |
| --- | --- | --- |
| **setLimit** | number | Sets the number of group members that can be fetched in a single request, suitable for pagination |
| **setSearchKeyword** | String | Used for fetching group members matching the passed string |
| **setScopes** | Array\<String> | Used for fetching group members based on multiple scopes |

In the example below, we filter group members by setting the limit to two and the scope to show only admin and moderator members.

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

<CometChatGroupMembers
  group={chatGroup}
  groupMemberRequestBuilder={
    new CometChat.GroupMembersRequestBuilder("GROUP_GUID")
      .setLimit(2)
      .setScopes(["admin", "moderator"])
  }
/>;
```

#### SearchRequestBuilder

The SearchRequestBuilder uses [GroupMembersRequestBuilder](/sdk/javascript/retrieve-group-members) to filter and customize the search list. This keeps uniformity between the displayed group members list and searched group members list.

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

<CometChatGroupMembers
  group={chatGroup}
  searchRequestBuilder={
    new CometChat.GroupMembersRequestBuilder("GROUP_GUID")
      .setLimit(2)
      .setSearchKeyword("**")
  }
/>;
```

### Events

[Events](/ui-kit/react/components-overview#events) are emitted by a `Component`. By using event you can extend existing functionality. Being global events, they can be applied in Multiple Locations and are capable of being Added or Removed.

Events emitted by the Group Members component:

| Event | Description |
| --- | --- |
| **ccGroupMemberBanned** | Triggers when a group member is banned from the group successfully |
| **ccGroupMemberKicked** | Triggers when a group member is kicked from the group successfully |
| **ccGroupMemberScopeChanged** | Triggers when a group member's scope is changed in the group |

Subscribe to group member events:

```tsx lines
import { CometChatGroupEvents } from "@cometchat/chat-uikit-react";

const ccGroupMemberBanned = CometChatGroupEvents.ccGroupMemberBanned.subscribe(
  (item: IGroupMemberKickedBanned) => {
    // Your code here
  }
);

const ccGroupMemberKicked = CometChatGroupEvents.ccGroupMemberKicked.subscribe(
  (item: IGroupMemberKickedBanned) => {
    // Your code here
  }
);

const ccGroupMemberScopeChanged =
  CometChatGroupEvents.ccGroupMemberScopeChanged.subscribe(
    (item: IGroupMemberScopeChanged) => {
      // Your code here
    }
  );
```

Unsubscribe from group member events:

```tsx lines
ccGroupMemberBanned?.unsubscribe();
ccGroupMemberKicked?.unsubscribe();
ccGroupMemberScopeChanged?.unsubscribe();
```

| Property | Description | Code |

| **Group** | The group for which members are being fetched. | `group={chatGroup}` |
| **Search Keyword** | The keyword used to filter the group members list. | `searchKeyword="admin"` |
| **Selection Mode** | Selection mode to use for the default list item view. | `selectionMode={SelectionMode.multiple}` |
| **Show Scrollbar** | Controls the visibility of the scrollbar in the list. | `showScrollbar={true}` |
| **Loading View** | A custom view to display during the loading state. | `loadingView={<LoadingSpinner />}` |
| **Error View** | Custom view for the error state of the component. | `errorView={<ErrorComponent />}` |
| **Empty View** | A custom view to display when no group members are available in the list. | `emptyView={<EmptyComponent />}` |

  CometChatUIKitLoginListener,

const GroupMembersDemo = () => {

          >
            {localize(scope.toUpperCase())}

    <>
      {chatGroup && (

        />
      )}
    </>
  );
};

export default GroupMembersDemo;

### Real-Time Group Member Events

The CometChatGroupMembers component emits events via `CometChatGroupEvents`. You can subscribe to these events to react to member changes across your application.

Subscribe to group member events:

```tsx lines
import { CometChatGroupEvents } from "@cometchat/chat-uikit-react";

// Subscribe
const subscription = CometChatGroupEvents.ccGroupMemberKicked.subscribe(
  (item: IGroupMemberKickedBanned) => {
    console.log("Member kicked:", item);
  }
);

// Unsubscribe when no longer needed (e.g., in useEffect cleanup)
subscription?.unsubscribe();
```

## Pagination, Batching, and Performance

The CometChatGroupMembers component fetches group members in pages using `GroupMembersRequestBuilder`. The default page size is 30 members per request.

### Customizing Page Size

Use `setLimit` on the `GroupMembersRequestBuilder` to control how many members are fetched per page:

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

<CometChatGroupMembers
  group={chatGroup}
  groupMemberRequestBuilder={
    new CometChat.GroupMembersRequestBuilder("GROUP_GUID").setLimit(15)
  }
/>;
```

### Infinite Scroll

The component uses infinite scroll by default. As the user scrolls to the bottom of the list, the next page of group members is fetched automatically. No additional configuration is required.

### Custom Loading View

You can provide a custom loading indicator that displays while group members are being fetched:

```tsx lines
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

<CometChatGroupMembers
  group={chatGroup}
  loadingView={<div className="custom-loader">Loading members...</div>}
/>;
```

## Error Handling and Retries

The CometChatGroupMembers component displays a default error view when member fetching fails. You can customize this behavior using the `onError`, `hideError`, and `errorView` props.

### Default Behavior

When an error occurs (such as a network failure or authentication issue), the component renders a built-in error state and logs the error to the console.

### onError Callback

Use the `onError` prop to intercept errors and implement custom error handling logic:

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

const handleOnError = (error: CometChat.CometChatException) => {
  console.error("Group members error:", error.message);
  // Send to your error tracking service
};

<CometChatGroupMembers group={chatGroup} onError={handleOnError} />;
```

### Hiding the Error View

Set `hideError` to `true` to suppress the default error UI entirely:

```tsx lines
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

<CometChatGroupMembers group={chatGroup} hideError={true} />;
```

### Custom Error View

Provide a custom error component using the `errorView` prop:

```tsx lines
import { CometChatGroupMembers } from "@cometchat/chat-uikit-react";

<CometChatGroupMembers
  group={chatGroup}
  errorView={
    <div className="custom-error">
      <p>Something went wrong. Please try again.</p>
    </div>
  }
/>;
```

## Configuration and Environment

### CometChat Initialization

The CometChatGroupMembers component requires `CometChatUIKit.init()` to be called before rendering. The initialization requires `appId`, `region`, and `authKey`:

```tsx lines
import { CometChatUIKit, UIKitSettingsBuilder } from "@cometchat/chat-uikit-react";

const settings = new UIKitSettingsBuilder()
  .setAppId("APP_ID")
  .setRegion("REGION")
  .setAuthKey("AUTH_KEY")
  .build();

await CometChatUIKit.init(settings);
await CometChatUIKit.login("USER_UID");
```

### Required Packages

The Group Members component requires only the core UI Kit package:

- `@cometchat/chat-uikit-react` — the React UI Kit containing all components
- `@cometchat/chat-sdk-javascript` — installed automatically as a peer dependency

### Supported React Versions and Frameworks

- For Next.js integration details, see [Next.js Integration](/ui-kit/react/next-js-integration)

## Logging, Tracing, and Observability

### Enabling Debug Logging

Enable CometChat SDK debug logging to troubleshoot group member fetching and real-time event issues:

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";

const appSettings = new CometChat.AppSettingsBuilder()
  .subscribePresenceForAllUsers()
  .setRegion("REGION")
  .autoEstablishSocketConnection(true)
  .build();

CometChat.init("APP_ID", appSettings).then(() => {
  CometChat.setLogLevel(CometChat.LogLevel.DEBUG);
});
```

### Observable Events

The component emits events through `CometChatGroupEvents` that can be used for observability:

- `ccGroupMemberBanned` — fired when a member is banned from the group
- `ccGroupMemberKicked` — fired when a member is kicked from the group
- `ccGroupMemberScopeChanged` — fired when a member's scope is changed

You can subscribe to these events to log user actions or feed analytics dashboards. See the [Events](#events) subsection in Basic Usage for subscribe/unsubscribe patterns.
