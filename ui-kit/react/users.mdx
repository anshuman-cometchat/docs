## Overview

The CometChatUsers component renders a searchable, scrollable list of all available users in your application. Each list item displays the user's avatar, name, and online/offline status indicator. It supports selection modes, custom view slots, filtering via `UsersRequestBuilder`, and CSS styling for building user directory and contact list experiences.

Import path: `import { CometChatUsers } from "@cometchat/chat-uikit-react";`

**Before using this component:** Ensure `CometChatUIKit.init(UIKitSettings)` has completed and the user is logged in via `CometChatUIKit.login("UID")`. See [React.js Integration](/ui-kit/react/react-js-integration).
</Note>

The Users component is composed of the following BaseComponents:

| Components        | Description                                                                                                     |
| ----------------- | --------------------------------------------------------------------------------------------------------------- |
| CometChatList     | A reusable container component having title, search box, customisable background and a list view.               |
| CometChatListItem | A component that renders data obtained from a User object on a Tile having a title, subtitle, leading and trailing view. |

## Why Use / When to Use

- Display a searchable directory of all registered users in your application
- Allow users to select a contact and navigate to a chat view with `onItemClick`
- Filter users by status, role, tags, or friends-only using `UsersRequestBuilder`
- Show online/offline presence indicators for each user in the list
- Customize the appearance of each user item using view slots like `itemView`, `subtitleView`, and `leadingView`
- Enable multi-select mode for batch operations such as creating group chats or sending broadcast messages
- Display alphabetical section headers to organize users by name

## Usage

### Actions

[Actions](/ui-kit/react/components-overview#actions) dictate how a component functions. They are divided into two types: Predefined and User-defined. You can override either type, allowing you to tailor the behavior of the component to fit your specific needs.

#### 1. OnItemClick

`OnItemClick` is triggered when you click on a user item in the list. This action does not have a predefined behavior. You can override this action using the following code snippet.

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

const handleOnItemClick = (user: CometChat.User) => {
  console.log("User clicked:", user.getName());
  // Your custom action here
};

<CometChatUsers onItemClick={handleOnItemClick} />;
```

#### 2. OnSelect

The `OnSelect` event is triggered upon the completion of a selection in `SelectionMode`. It does not have a default behavior. However, you can override its behavior using the following code snippet.

```tsx lines
import { CometChatUsers, SelectionMode } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

const handleOnSelect = (user: CometChat.User, selected: boolean) => {
  console.log("Selected:", user.getName(), selected);
  // Your custom action on select
};

<CometChatUsers
  selectionMode={SelectionMode.multiple}
  onSelect={handleOnSelect}
/>;
```

#### 3. OnEmpty

This action allows you to specify a callback function to be executed when the user list is empty.

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";

const handleOnEmpty = () => {
  console.log("No users available");
};

<CometChatUsers onEmpty={handleOnEmpty} />;
```

#### 4. OnError

This action listens for any errors that occur in the Users component.

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

const handleOnError = (error: CometChat.CometChatException) => {
  console.error("Users error:", error.message);
  // Your exception handling code
};

<CometChatUsers onError={handleOnError} />;
```

### Filters

You can set `UsersRequestBuilder` in the Users component to filter the user list. You can modify the builder as per your specific requirements with multiple options available to know more refer to [UsersRequestBuilder](/sdk/javascript/retrieve-users).

You can set filters using the following parameters:

1. **Limit:** Number of users fetched in a single request.
2. **SearchKeyword:** Fetches users matching the passed string.
3. **HideBlockedUsers:** Filters out users blocked by the logged-in user.
4. **FriendsOnly:** Fetches only friends of the logged-in user.
5. **Roles:** Filters users by specific roles.
6. **Tags:** Filters users by specific tags.
7. **Status:** Filters users by online or offline status.
8. **UIDs:** Fetches specific users by their UIDs.

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

<CometChatUsers
  usersRequestBuilder={
    new CometChat.UsersRequestBuilder().friendsOnly(true)
  }
/>;
```

### Events

[Events](/ui-kit/react/components-overview#events) are emitted by a `Component`. By using event you can extend existing functionality. Being global events, they can be applied in Multiple Locations and are capable of being Added or Removed.

The `Users` component does not produce any events directly. However, you can listen for user presence changes using the CometChat SDK:

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";

const listenerID = "USERS_PRESENCE_LISTENER";

// Subscribe to user presence events
CometChat.addUserListener(
  listenerID,
  new CometChat.UserListener({
    onUserOnline: (onlineUser: CometChat.User) => {
      console.log("User online:", onlineUser.getName());
    },
    onUserOffline: (offlineUser: CometChat.User) => {
      console.log("User offline:", offlineUser.getName());
    },
  })
);

// Unsubscribe when no longer needed (e.g., in useEffect cleanup)
CometChat.removeUserListener(listenerID);
```

### Real-Time User Presence Events

The CometChatUsers component automatically reflects user presence changes. You can also subscribe to presence events directly using the CometChat SDK to react to user status changes across your application.

Subscribe to user presence events:

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";

const listenerID = "USERS_PRESENCE_LISTENER";

// Subscribe
CometChat.addUserListener(
  listenerID,
  new CometChat.UserListener({
    onUserOnline: (onlineUser: CometChat.User) => {
      console.log("User came online:", onlineUser.getName());
    },
    onUserOffline: (offlineUser: CometChat.User) => {
      console.log("User went offline:", offlineUser.getName());
    },
  })
);

// Unsubscribe when no longer needed (e.g., in useEffect cleanup)
CometChat.removeUserListener(listenerID);
```

## Pagination, Batching, and Performance

The CometChatUsers component fetches users in pages using `UsersRequestBuilder`. The default page size is 30 users per request.

### Customizing Page Size

Use `setLimit` on the `UsersRequestBuilder` to control how many users are fetched per page:

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

<CometChatUsers
  usersRequestBuilder={
    new CometChat.UsersRequestBuilder().setLimit(15)
  }
/>;
```

### Infinite Scroll

The component uses infinite scroll by default. As the user scrolls to the bottom of the list, the next page of users is fetched automatically. No additional configuration is required.

### Custom Loading View

You can provide a custom loading indicator that displays while users are being fetched:

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";

<CometChatUsers
  loadingView={<div className="custom-loader">Loading users...</div>}
/>;
```

| Methods | Type | Description |

| **setLimit** | number | Sets the number of users that can be fetched in a single request, suitable for pagination |
| **setSearchKeyword** | String | Used for fetching users matching the passed string |
| **hideBlockedUsers** | boolean | Used for fetching all those users who are not blocked by the logged-in user |
| **friendsOnly** | boolean | Used for fetching only those users in which logged-in user is a friend |

## Error Handling and Retries

The CometChatUsers component displays a default error view when user fetching fails. You can customize this behavior using the `onError`, `hideError`, and `errorView` props.

### Default Behavior

When an error occurs (such as a network failure or authentication issue), the component renders a built-in error state and logs the error to the console.

### onError Callback

Use the `onError` prop to intercept errors and implement custom error handling logic:

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";
import { CometChat } from "@cometchat/chat-sdk-javascript";

const handleOnError = (error: CometChat.CometChatException) => {
  console.error("Users error:", error.message);
  // Send to your error tracking service
};

<CometChatUsers onError={handleOnError} />;
```

### Hiding the Error View

Set `hideError` to `true` to suppress the default error UI entirely:

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";

<CometChatUsers hideError={true} />;
```

### Custom Error View

Provide a custom error component using the `errorView` prop:

```tsx lines
import { CometChatUsers } from "@cometchat/chat-uikit-react";

<CometChatUsers
  errorView={
    <div className="custom-error">
      <p>Something went wrong. Please try again.</p>
    </div>
  }
/>;
```

## Configuration and Environment

### CometChat Initialization

The CometChatUsers component requires `CometChatUIKit.init()` to be called before rendering. The initialization requires `appId`, `region`, and `authKey`:

```tsx lines
import { CometChatUIKit, UIKitSettingsBuilder } from "@cometchat/chat-uikit-react";

const settings = new UIKitSettingsBuilder()
  .setAppId("APP_ID")
  .setRegion("REGION")
  .setAuthKey("AUTH_KEY")
  .build();

await CometChatUIKit.init(settings);
await CometChatUIKit.login("USER_UID");
```

### Required Packages

The Users component requires only the core UI Kit package:

- `@cometchat/chat-uikit-react` — the React UI Kit containing all components
- `@cometchat/chat-sdk-javascript` — installed automatically as a peer dependency

### Supported React Versions and Frameworks

- React 18+ is supported
- Compatible with Next.js (wrap in `useEffect` or use dynamic imports with `ssr: false` to avoid SSR hydration errors), React Router, and Astro
- For Next.js integration details, see [Next.js Integration](/ui-kit/react/next-js-integration)

## Files and Media

Files and media handling does not apply to the Users component. The user list displays user profiles but does not render or send media directly. For file and media support, see the [Message Composer](/ui-kit/react/message-composer) and [Message List](/ui-kit/react/message-list) components.

## Logging, Tracing, and Observability

### Enabling Debug Logging

Enable CometChat SDK debug logging to troubleshoot user fetching and presence event issues:

```tsx lines
import { CometChat } from "@cometchat/chat-sdk-javascript";

const appSettings = new CometChat.AppSettingsBuilder()
  .subscribePresenceForAllUsers()
  .setRegion("REGION")
  .autoEstablishSocketConnection(true)
  .build();

CometChat.init("APP_ID", appSettings).then(() => {
  CometChat.setLogLevel(CometChat.LogLevel.DEBUG);
});
```

### Observable Events

The Users component reflects real-time presence changes via the CometChat SDK's user listeners. You can subscribe to `onUserOnline` and `onUserOffline` events to log user activity or feed analytics dashboards. See the [Events](#events) subsection in Basic Usage and the [Real-Time User Presence Events](#real-time-user-presence-events) section in Advanced Features for subscribe/unsubscribe patterns.
