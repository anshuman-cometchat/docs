## Overview

This guide demonstrates how to implement calling functionality using only the CometChat Calls SDK, without requiring the Chat SDK. This is ideal for applications that need video/audio calling capabilities without the full chat infrastructure.

<Note>
Before you begin, ensure you have completed the [Calls SDK setup](/sdk/android/setup-calling).
</Note>

## User Authentication

To start a call session, you need a user auth token. Since this implementation doesn't use the Chat SDK, you'll need to obtain the auth token via the CometChat REST API.

<Note>
To understand user authentication in CometChat, see the [User Auth](/fundamentals/user-auth) guide.
</Note>

You can obtain the auth token using one of these REST API endpoints:

- [Create Auth Token](/rest-api/auth-tokens/create) — Creates a new auth token for a user
- [Get Auth Token](/rest-api/auth-tokens/get) — Retrieves an existing auth token

Store the auth token securely in your application for use when generating call tokens.

## Generate Call Token

A call token is required for secure access to a call session. Each token is unique to a specific session and user combination, ensuring that only authorized users can join the call.

You can generate the token just before starting the call, or generate and store it ahead of time based on your use case.

Use the `generateToken()` method to create a call token:

<Tabs>
<Tab title="Java">
```java
String sessionId = "UNIQUE_SESSION_ID"; // Generate a unique session ID
String userAuthToken = "USER_AUTH_TOKEN"; // Obtained from REST API

CometChatCalls.generateToken(sessionId, userAuthToken, new CometChatCalls.CallbackListener<GenerateToken>() {
    @Override
    public void onSuccess(GenerateToken generateToken) {
        // Use generateToken to start the session
    }

    @Override
    public void onError(CometChatException e) {
        Log.e(TAG, "Token generation failed: " + e.getMessage());
    }
});
```

</Tab>

<Tab title="Kotlin">
```kotlin
val sessionId = "UNIQUE_SESSION_ID" // Generate a unique session ID
val userAuthToken = "USER_AUTH_TOKEN" // Obtained from REST API

CometChatCalls.generateToken(sessionId, userAuthToken, object : CometChatCalls.CallbackListener<GenerateToken>() {
    override fun onSuccess(generateToken: GenerateToken) {
        // Use generateToken to start the session
    }

    override fun onError(e: CometChatException) {
        Log.e(TAG, "Token generation failed: ${e.message}")
    }
})
```

</Tab>

</Tabs>

| Parameter       | Description                                                                |
| --------------- | -------------------------------------------------------------------------- |
| `sessionId`     | A unique session ID for the call. Generate this yourself or use a shared ID for participants to join the same call. |
| `userAuthToken` | The user auth token obtained from the CometChat REST API.                  |

## Start Call Session

Use the `startSession()` method to join a call session. This method requires a call token (generated in the previous step) and a `CallSettings` object that configures the call UI and behavior.

The `CallSettings` class configures the call UI and behavior. Use `CallSettingsBuilder` to create a `CallSettings` instance with the following required parameters:

| Parameter          | Description                                                              |
| ------------------ | ------------------------------------------------------------------------ |
| `activityContext`  | The activity context where the call UI will be displayed                 |
| `videoContainer`   | A `RelativeLayout` where the calling UI will be rendered                 |
| `callToken`        | The `GenerateToken` object received from `generateToken()` onSuccess     |

<Tabs>
<Tab title="Java">
```java
RelativeLayout videoContainer = findViewById(R.id.video_container);
Context activityContext = this;
GenerateToken callToken; // Received from generateToken() onSuccess

CallSettings callSettings = new CometChatCalls.CallSettingsBuilder(activityContext, videoContainer)
    .setDefaultLayoutEnable(true)
    .setIsAudioOnly(false)
    .build();

CometChatCalls.startSession(callToken, callSettings, new CometChatCalls.CallbackListener<String>() {
    @Override
    public void onSuccess(String s) {
        // Call session started successfully
    }

    @Override
    public void onError(CometChatException e) {
        // Handle error
    }
});
```

</Tab>

<Tab title="Kotlin">
```kotlin
val videoContainer: RelativeLayout = findViewById(R.id.video_container)
val activityContext: Context = this
val callToken: GenerateToken // Received from generateToken() onSuccess

        // Call session started successfully
    }

        // Handle error
    }

```

### Call Settings

Configure the call experience using the following `CallSettingsBuilder` methods:

<AccordionGroup>
  <Accordion title="setDefaultLayoutEnable(boolean)">
    Enables or disables the default call UI layout with built-in controls.
    
    - `true` — Shows the default layout with end call, mute, video toggle buttons
    - `false` — Hides the button layout, displays only the call view
    
    **Default:** `true`
  </Accordion>

  <Accordion title="setIsAudioOnly(boolean)">
    Sets whether the call is audio-only or audio-video.
    
    - `true` — Audio-only call (no video)
    - `false` — Audio-video call
    
    **Default:** `false`
  </Accordion>

  <Accordion title="setEventListener(CometChatCallsEventsListener)">
    Sets the listener to receive call events. See [Call Listeners](#call-listeners) for available callbacks.
  </Accordion>

  <Accordion title="setMode(String)">
    Sets the call UI layout mode.
    
    **Available modes:**
    - `CometChatCallsConstants.MODE_DEFAULT` — Grid layout showing all participants
    - `CometChatCallsConstants.MODE_SPOTLIGHT` — Highlights the active speaker
    - `CometChatCallsConstants.MODE_SINGLE` — Shows only one participant at a time
    
    **Default:** `MODE_DEFAULT`
  </Accordion>

  <Accordion title="setAvatarMode(String)">
    Sets how participant avatars are displayed when video is off.
    
    **Available modes:**
    - `CometChatCallsConstants.AVATAR_MODE_CIRCLE` — Circular avatar
    - `CometChatCallsConstants.AVATAR_MODE_SQUARE` — Square avatar
    - `CometChatCallsConstants.AVATAR_MODE_FULLSCREEN` — Fullscreen avatar
    
    **Default:** `AVATAR_MODE_CIRCLE`
  </Accordion>

  <Accordion title="setDefaultAudioMode(String)">
    Sets the initial audio output device when the call starts.
    
    **Available modes:**
    - `CometChatCallsConstants.AUDIO_MODE_SPEAKER` — Device speaker
    - `CometChatCallsConstants.AUDIO_MODE_EARPIECE` — Phone earpiece
    - `CometChatCallsConstants.AUDIO_MODE_BLUETOOTH` — Bluetooth device
    - `CometChatCallsConstants.AUDIO_MODE_HEADPHONES` — Wired headphones
  </Accordion>

  <Accordion title="startWithAudioMuted(boolean)">
    Starts the call with the microphone muted.
    
    **Default:** `false`
  </Accordion>

  <Accordion title="startWithVideoMuted(boolean)">
    Starts the call with the camera turned off.
    
    **Default:** `false`
  </Accordion>

  <Accordion title="showEndCallButton(boolean)">
    Shows or hides the end call button in the default layout.
    
    **Default:** `true`
  </Accordion>

  <Accordion title="showSwitchCameraButton(boolean)">
    Shows or hides the switch camera button (front/back) in the default layout.
    
    **Default:** `true`
  </Accordion>

  <Accordion title="showMuteAudioButton(boolean)">
    Shows or hides the mute audio button in the default layout.
    
    **Default:** `true`
  </Accordion>

  <Accordion title="showPauseVideoButton(boolean)">
    Shows or hides the pause video button in the default layout.
    
    **Default:** `true`
  </Accordion>

  <Accordion title="showAudioModeButton(boolean)">
    Shows or hides the audio mode selection button (speaker/earpiece/bluetooth) in the default layout.
    
    **Default:** `true`
  </Accordion>

  <Accordion title="showSwitchToVideoCallButton(boolean)">
    Shows or hides the button to upgrade an audio call to a video call.
    
    **Default:** `true`
  </Accordion>

  <Accordion title="showRecordingButton(boolean)">
    Shows or hides the recording button in the default layout.
    
    **Default:** `false`
  </Accordion>

  <Accordion title="autoRecordOnCallStart(boolean)">
    Automatically starts recording when the call begins.
    
    **Default:** `false`
  </Accordion>

  <Accordion title="enableVideoTileClick(boolean)">
    Enables or disables click interactions on video tiles in Spotlight mode.
    
    **Default:** `true`
  </Accordion>

  <Accordion title="enableVideoTileDrag(boolean)">
    Enables or disables drag functionality for video tiles in Spotlight mode.
    
    **Default:** `true`
  </Accordion>

  <Accordion title="setMainVideoContainerSetting(MainVideoContainerSetting)">
    Customizes the main video container appearance and behavior. See [Video View Customization](/sdk/android/video-view-customisation) for details.
  </Accordion>

  <Accordion title="setIdleTimeoutPeriod(int)">
    Sets the idle timeout duration in seconds. If the user remains the only participant for this duration, a warning appears 60 seconds before the call auto-terminates.
    
    **Default:** `180` seconds
    
    *Available since v4.1.0*
  </Accordion>
</AccordionGroup>

## Call Listeners

The `CometChatCallsEventsListener` interface provides real-time callbacks for call session events, including participant changes, call state updates, and error conditions. Register this listener in any activity or fragment where you need to respond to call events.

Each listener requires a unique `listenerId` string. This ID is used to:
- **Prevent duplicate registrations** — Re-registering with the same ID replaces the existing listener
- **Enable targeted removal** — Remove specific listeners without affecting others

<Tabs>
<Tab title="Java">
```java
private String listenerId = "UNIQUE_LISTENER_ID";

// Register listener (e.g., in onCreate or onResume)
CometChatCalls.addCallsEventListeners(listenerId, new CometChatCallsEventsListener() {
    @Override
    public void onCallEnded() {
        // Call ended, close the calling screen
    }

    @Override
    public void onSessionTimeout() {
        // Session timed out due to inactivity
    }

    @Override
    public void onCallEndButtonPressed() {
        // User pressed end call button
        CometChatCalls.endSession();
    }

    @Override
    public void onUserJoined(RTCUser user) {
        // A participant joined the call
    }

    @Override
    public void onUserLeft(RTCUser user) {
        // A participant left the call
    }

    @Override
    public void onUserListChanged(ArrayList<RTCUser> users) {
        // Participant list updated
    }

    @Override
    public void onAudioModeChanged(ArrayList<AudioMode> devices) {
        // Available audio devices changed
    }

    @Override
    public void onCallSwitchedToVideo(CallSwitchRequestInfo info) {
        // Call upgraded from audio to video
    }

    @Override
    public void onUserMuted(RTCMutedUser muteObj) {
        // A participant's mute state changed
    }

    @Override
    public void onRecordingToggled(RTCRecordingInfo info) {
        // Recording started or stopped
    }

    @Override
    public void onError(CometChatException ce) {
        // An error occurred
    }
});

// Unregister listener (e.g., in onDestroy or onPause)
CometChatCalls.removeCallsEventListeners(listenerId);
```

</Tab>

<Tab title="Kotlin">
```kotlin
val listenerId = "UNIQUE_LISTENER_ID"

    }

// Unregister listener (e.g., in onDestroy or onPause)
CometChatCalls.removeCallsEventListeners(listenerId)
```

### Events

<AccordionGroup>
  <Accordion title="onCallEnded()">
    Invoked when the call session terminates for a 1:1 call. Both participants receive this callback regardless of who initiated the termination.
    
    **Behavior:**
    - Only fires for calls with exactly 2 participants
    - Does not fire for group calls or when a single user is in the session
    - The user who ended the call also receives `onCallEndButtonPressed()`
    
    **Use case:** Clean up UI resources, navigate away from the call screen.
  </Accordion>
  
  <Accordion title="onSessionTimeout()">
    Invoked when the call session is automatically terminated due to inactivity or exceeding the configured idle timeout period.
    
    **Behavior:**
    - Triggered after the timeout duration set via `setIdleTimeoutPeriod()` (default: 180 seconds)
    - A warning prompt appears 60 seconds before termination
    
    **Use case:** Handle automatic session cleanup, notify the user about timeout.
    
    *Available since v4.1.0*
  </Accordion>
  
  <Accordion title="onCallEndButtonPressed()">
    Invoked when the local user taps the end call button in the calling UI.
    
    **Behavior:**
    - Only the user who pressed the button receives this callback
    - Call `CometChatCalls.endSession()` to leave the session
    
    **Use case:** Trigger call termination logic, differentiate between user-initiated and remote-initiated call endings.
  </Accordion>
  
  <Accordion title="onUserJoined(user: RTCUser)">
    Invoked when a remote participant joins the call session.
    
    **Parameters:**
    - `user`: `RTCUser` object containing participant details (UID, name, avatar)
    
    **Use case:** Update participant list UI, show join notifications, adjust video grid layout.
  </Accordion>
  
  <Accordion title="onUserLeft(user: RTCUser)">
    Invoked when a remote participant leaves the call session.
    
    **Parameters:**
    - `user`: `RTCUser` object of the participant who left
    
    **Use case:** Update participant list UI, show leave notifications, handle last-participant scenarios.
  </Accordion>
  
  <Accordion title="onUserListChanged(users: ArrayList)">
    Invoked whenever the participant list changes (join or leave events).
    
    **Parameters:**
    - `users`: List of `RTCUser` objects representing all current participants
    
    **Use case:** Sync local participant state with the server, rebuild participant grid, update participant count.
  </Accordion>
  
  <Accordion title="onAudioModeChanged(devices: ArrayList)">
    Invoked when available audio output devices change (e.g., Bluetooth connected/disconnected).
    
    **Parameters:**
    - `devices`: List of `AudioMode` objects representing available outputs
    
    **Available modes:**
    - `AUDIO_MODE_SPEAKER` - Device speaker
    - `AUDIO_MODE_EARPIECE` - Phone earpiece
    - `AUDIO_MODE_BLUETOOTH` - Bluetooth device
    - `AUDIO_MODE_HEADPHONES` - Wired headphones
    
    **Use case:** Update audio routing UI, auto-switch to preferred device.
  </Accordion>
  
  <Accordion title="onCallSwitchedToVideo(info: CallSwitchRequestInfo)">
    Invoked when an audio call is upgraded to a video call.
    
    **Parameters:**
    - `info`: `CallSwitchRequestInfo` object containing switch request details
    
    **Use case:** Update UI to show video controls, request camera permissions if needed.
  </Accordion>
  
  <Accordion title="onUserMuted(muteObj: RTCMutedUser)">
    Invoked when a participant's mute state changes.
    
    **Parameters:**
    - `muteObj`: `RTCMutedUser` object containing the user and their mute status
    
    **Use case:** Update mute indicators in participant list, show mute/unmute notifications.
  </Accordion>
  
  <Accordion title="onRecordingToggled(info: RTCRecordingInfo)">
    Invoked when call recording starts or stops.
    
    **Parameters:**
    - `info`: `RTCRecordingInfo` object containing recording state and metadata
    
    **Use case:** Show recording indicator, notify participants about recording status.
  </Accordion>
  
  <Accordion title="onError(e: CometChatException)">
    Invoked when an error occurs during the call session.
    
    **Parameters:**
    - `e`: `CometChatException` containing error code and message
    
    **Common errors:** Network failures, permission issues, server errors
    
    **Use case:** Display error messages, implement retry logic, log errors for debugging.
  </Accordion>
</AccordionGroup>

## End Call Session

To end the call session and release all media resources (camera, microphone, network connections), call `CometChatCalls.endSession()` in the `onCallEndButtonPressed()` callback.

<Tabs>
<Tab title="Java">
```java
// In onCallEndButtonPressed() callback
CometChatCalls.endSession();
// Close the calling screen
```

</Tab>

<Tab title="Kotlin">
```kotlin
// In onCallEndButtonPressed() callback
CometChatCalls.endSession()
// Close the calling screen
```

</Tab>

</Tabs>

## Methods

These methods are available for performing custom actions during an active call session. Use them to build custom UI controls or implement specific behaviors based on your use case.

<Note>
These methods can only be called when a call session is active.
</Note>

### Mute Audio

Controls the local audio stream transmission. When muted, other participants cannot hear the local user.

- `false` — Unmutes the microphone, resumes audio transmission

- `CometChatCallsConstants.AUDIO_MODE_EARPIECE` — Phone earpiece
- `CometChatCallsConstants.AUDIO_MODE_BLUETOOTH` — Connected Bluetooth device
- `CometChatCallsConstants.AUDIO_MODE_HEADPHONES` — Wired headphones
