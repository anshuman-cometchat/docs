import { Steps, Step } from 'mintlify';

You can add the CometChat Widget to any page by pasting a short HTML snippet. Think of it as a ready-made chat window—drop it where you want it, then decide how people should get into the chat. Pick the option below that feels closest to your website:

<Steps className="mt-6">
  <Step title="1. Guest / Anonymous Users (No Login)">
    <p>Use this when you want visitors to chat right away on a marketing page, help center, or event site. CometChat uses the Guest Auth Key from your dashboard to spin up a temporary profile, and you can still set a friendly display name or avatar so messages stay clear.</p>
  </Step>

  <Step title="2. Create User on the Fly (UID Login)">
    <p>Choose this if you already give each customer a unique ID (maybe an email, customer number, or username). Drop that ID into the config and CometChat creates the user the first time they open chat. They’ll see the same name, avatar, and conversation history every time they return—no extra backend work needed.</p>
  </Step>

  <Step title="3. App With Its Own Backend (Auth Token Login)">
    <p>Pick this when your app already has its own login and a backend team. After a user signs in, your server requests a CometChat auth token for them. The widget uses that token to log in, which keeps your secret keys off the browser and lets you control who can chat and for how long.</p>
  </Step>
</Steps>

## 1. Guest / Anonymous Users (No Login)

**Use this when:**

- Any visitor should be able to start chatting instantly  
- You do not have (or do not want to wire up) your own login/registration yet  

### 1.1 Code for `<head>`

```html
<script defer src="https://cdn.jsdelivr.net/npm/@cometchat/chat-embed@1.x.x/dist/main.js"></script>
```

### 1.2 Code for `<body>`

```html lines highlight={9-10, 12, 16-18, 23-25}
<!-- Where the chat widget should appear -->
<div id="cometChatMount"></div>

<script type="module">
  import { CometChatAuth } from "https://cdn.example.com/cometchat/cometchat-auth.js";

  //  EDIT ONLY THE TEXT INSIDE QUOTES
  const COMETCHAT_WIDGET_CONFIG = {
    appID: "YOUR_APP_ID",
    region: "YOUR_APP_REGION",   // e.g. "us", "eu", "in"
    mode: "guest",               // keep this as "guest"
    authKey: "YOUR_AUTH_KEY",

    // Optional: how the guest appears in chat
    user: {
      name: "Guest User",
      avatar: "",                // e.g. "https://example.com/guest.png"
      link: ""                   // e.g. "https://example.com/profile"
    },

    // Widget placement + size
    mount: "#cometChatMount",
    width: "450px",
    height: "80vh",
    isDocked: true               // true = docked widget (recommended)
  };

  CometChatAuth.start(COMETCHAT_WIDGET_CONFIG);
</script>
```

## 2. Create User on the Fly (UID Login)

**Use this when:**

- Your application already has unique user IDs  
- You want every user to appear with a stable identity in chat  

### 2.1 Code for `<head>`

```html
<script defer src="https://cdn.jsdelivr.net/npm/@cometchat/chat-embed@1.x.x/dist/main.js"></script>
```

### 2.2 Code for `<body>`

```html lines highlight={9-10, 12, 15, 19-21, 26-28}
<!-- Where the chat widget should appear -->
<div id="cometChatMount"></div>

<script type="module">
  import { CometChatAuth } from "https://cdn.example.com/cometchat/cometchat-auth.js";

  //  EDIT ONLY THE TEXT INSIDE QUOTES
  const COMETCHAT_WIDGET_CONFIG = {
    appID: "YOUR_APP_ID",
    region: "YOUR_APP_REGION",      // e.g. "us", "eu", "in"
    mode: "uid",                    // keep this as "uid"
    authKey: "YOUR_AUTH_KEY",

    // IMPORTANT: this must be unique per user
    uid: "UNIQUE_USER_ID",          // e.g. "user_123", "customer_42"

    // Optional: user profile shown in chat
    user: {
      name: "User Name",            // e.g. "Alex Johnson"
      avatar: "https://example.com/uid.png",
      link: "https://example.com/profile/uid"
    },

    // Widget placement + size
    mount: "#cometChatMount",
    width: "450px",
    height: "80vh",
    isDocked: true
  };

  CometChatAuth.start(COMETCHAT_WIDGET_CONFIG);
</script>
```

## 3. App With Its Own Backend (Auth Token Login)

**Use this when:**

- Your users already sign in through your backend  
- Your backend generates CometChat auth tokens for logged-in users  

### 3.1 Code for `<head>`

```html
<script defer src="https://cdn.jsdelivr.net/npm/@cometchat/chat-embed@1.x.x/dist/main.js"></script>
```

### 3.2 Code for `<body>`

```html lines highlight={9-10, 14, 18-19}
<!-- Where the chat widget should appear -->
<div id="cometChatMount"></div>

<script type="module">
  import { CometChatAuth } from "https://cdn.example.com/cometchat/cometchat-auth.js";

  //  EDIT ONLY THE TEXT INSIDE QUOTES
  const COMETCHAT_WIDGET_CONFIG = {
    appID: "YOUR_APP_ID",
    region: "YOUR_APP_REGION",       // e.g. "us", "eu", "in"
    mode: "authToken",               // keep this as "authToken"

    // Generated BY YOUR BACKEND after user login
    authToken: "USER_AUTH_TOKEN",

    // Widget placement + size
    mount: "#cometChatMount",
    width: "450px",
    height: "80vh",
    isDocked: true
  };

  CometChatAuth.start(COMETCHAT_WIDGET_CONFIG);
</script>
```
